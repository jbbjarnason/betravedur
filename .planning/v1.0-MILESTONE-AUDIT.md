## Cross-Phase Integration — End-to-End Data Path

The critical data path from Veðurstofan API to rendered map marker was verified continuous:

```
api.vedur.is
  → fetchAwsDay/fetchSynopDay (schema-assert + normalize)     [packages/fetch]
  → backfillStation (chunked, paced, resumable)               [pipeline/src/backfill.ts]
  → upsertPartition (field-pruned NDJSON, 10 fields)          [pipeline/src/rawstore.ts]
  → aggregateStation (groupBySeasonYear + encodeDerived)       [pipeline/src/aggregate.ts]
  → derived/{station}.{hash}.json on data branch              [git data branch]
  → manifest.json (content-hashed filenames)                  [pipeline/src/manifest.ts]
  → loadManifest + loadDerived (boot fetch, BASE_URL-aware)   [site/src/data/load.ts]
  → decodeDerived + domain math (recompute on cache)          [site/src/data/averages.ts]
  → MarkerDatum[] (score, temp, wind, precip, N, missingRain) [site/src/data/types.ts]
  → buildPill (scoreColor, °C, wind arrow, precip glyph)      [site/src/map/markers.ts]
  → #marker-overlay DOM pills on MapLibre collision survivors  [rendered site]
```

No orphaned links found. All wiring verified through Phase 1–8 verifications and confirmed live with E2E tests asserting `dataRequests===0` on recompute (data from boot cache, not re-fetched).

## Known Limitations (Correct-by-Design, Not Gaps)

1. **2-station sample vs full national dataset.** The `data` branch holds real history for 2 stations (#1 SYNOP Reykjavík 1949–2026 and #1350 AWS Keflavik 2008–2026). The full national enumeration is one operator-triggered `workflow_dispatch` away (`full_backfill: true`). The enumeration code is wired (`stations-list.ts`) and the PIPELINE.md §8(c) documents the two human prerequisites: push repo to GitHub and set Pages source = GitHub Actions.

2. **Precipitation limited to ~8 SYNOP stations.** AWS stations lack rain sensors (`r: null`). The score renormalizes over available components and displays "án úrkomu" badge. This is a Veðurstofan data-availability constraint, documented in research/01-CONTEXT.md and in the transparent score explainer.

3. **Basemap renders both "ICELAND" and "Ísland" labels** despite `lang:"is"`. This is a Protomaps grayscale-flavor limitation noted in STATE.md Pending Todos — non-blocking cosmetic issue for a future basemap refresh.

4. **Two human one-time setup steps** required before the nightly automation runs in production: (a) push repo to GitHub (triggering the data-branch first push from the workflow), (b) set Pages source = GitHub Actions. Both documented in PIPELINE.md §8.
