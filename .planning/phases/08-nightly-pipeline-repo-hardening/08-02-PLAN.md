<yaml_shape>
<!-- Reference shape from RESEARCH Code Examples. Executor refines; the test below is the gate. -->
- name: nightly
- on.schedule.cron: "37 4 * * *"  (off-peak, NOT 00:00); on.workflow_dispatch.inputs.full_backfill (boolean, default false)
- concurrency: group betravedur-data-branch, cancel-in-progress: false
- permissions (top-level or per-job): contents: write, pages: write, id-token: write
- job ingest-build (ubuntu-latest), outputs.changed:
  1. actions/checkout@v4 (main, read-only)
  2. actions/setup-node@v4 (node 22, cache npm)
  3. npm ci
  4. npm test && npm run typecheck   (GATE — fail stops the run before any push/deploy)
  5. materialize data branch: git fetch origin data --depth=1; git worktree add ./data-wt data
  6. backfill: if full_backfill -> enumerate national set + backfill each; else resume known stations, all with --root ./data-wt
  7. aggregate: npm run aggregate -- --root ./data-wt <specs>   (touched-only)
  8. commit delta in ./data-wt: git status --porcelain gate; if changes -> add/commit/push origin data, set changed=true; else changed=false (skip-empty, no empty commits)
  9. stage ship-set into site/public/data via copyShipSet (raw/ never copied)
  10. npm --prefix site run build; actions/upload-pages-artifact@v3 (path site/dist)
  11. heartbeat: if success() and secrets.HEARTBEAT_URL set -> curl -fsS -m 10 "$HEARTBEAT_URL" || true  (never fails run)
- job deploy: needs ingest-build; if changed=='true' || workflow_dispatch; concurrency group pages; permissions pages+id-token; environment github-pages; actions/deploy-pages@v4
- FORCE-PUSH: nightly push is a plain `git push origin data` (fast-forward). Any `--force` lives ONLY in the squash-reset (Plan 03), scoped to `origin data`. No workflow step ever pushes/force-pushes main.
</yaml_shape>
</context>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner → git remote | the job pushes to `data`; a mis-scoped push could corrupt main/live |
| Workflow → secrets (HEARTBEAT_URL, GITHUB_TOKEN) | secrets must never be echoed |
| Marketplace actions → runner | supply-chain surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-10 | Tampering | force-push clobbers main/live site | mitigate | Nightly push is fast-forward `git push origin data` only; workflow.test asserts no `push ... main` and no `--force` in nightly.yml; all writes scoped to ./data-wt worktree |
| T-08-11 | DoS | unbounded .git from nightly commits | mitigate | skip-empty (touched-only aggregate → no diff → no commit); squash-reset cadence in Plan 03 |
| T-08-12 | Info Disclosure | heartbeat URL / token leakage | mitigate | HEARTBEAT_URL via secrets, curl `-fsS` (no `-v`), never printed; GITHUB_TOKEN auto-provided, least-privilege permissions block |
| T-08-13 | Elevation | over-broad workflow permissions | mitigate | Explicit least-privilege `permissions:` (contents:write for data push, pages+id-token for deploy); deploy job restricts to pages+id-token |
| T-08-SC | Tampering | unpinned marketplace actions | mitigate | Pin actions/* to major tags (checkout@v4, setup-node@v4, upload-pages-artifact@v3, deploy-pages@v4); RESEARCH Package Legitimacy Audit: all first-party, approved |
</threat_model>
