<objective>
Build the resumable backfill runner and the partitioned raw store — the DATA-02 engine that turns paced API calls into a field-pruned, idempotent, re-derivable NDJSON store on disk. Encode the live-measured API reality directly: chunk to <=5 station-years, pace <=4 req/s, halve on 413, retry 502/503, treat 404 as empty, and never burst-parallelize (503 throttle). Persist only the 10 `DailyObservation` fields so the raw store stays ~386 MB uncompressed (DATA-07), not ~2 GB.

Purpose: This is the one-time backfill that makes 2010-2015-style ranges (and far deeper) available, and the re-derivation source of truth that lets `aggregate` (Plan 04) rebuild derived files without re-hitting the API. Its idempotency and resumability are what Phase 8's nightly cron will lean on.

Output: `backfill.ts` (chunk loop + high-water resume), `rawstore.ts` (upsert-by-(station,date) partitions), a status-aware `fetchWithRetry` extension, and offline mock-fetch tests covering the full 413/502/503/404 error taxonomy + idempotency + resume.
</objective>

# Measured API error taxonomy (02-RESEARCH, live 2026-07-19) — DO NOT re-derive
| Status | Meaning | Loop action |
|--------|---------|-------------|
| 413 | "Too many parameters…" (size ceiling, ~77yr) | Do NOT retry as-is; halve span (5yr->2-3yr->1yr; give up a single year after depth 3) |
| 502 | Bad Gateway (flaky ~20k-row zone) | Retry with backoff; if persistent, halve |
| 503 | Throttle (burst concurrency) | Back off longer; retry; NEVER treat as empty |
| 5xx/network | transient | existing exponential backoff |
| 404 {message} | no data | NOT an error: `[]` already; record empty chunk, advance |
| 422 {detail} | bad param | throw (programming error) |

# Locked policy (02-CONTEXT / 02-RESEARCH)
- Chunk size default: 1 station × 5 years (~1826 rows). Fallback on 413: 1 station-year (365 rows).
- Pace: sequential, ~250ms sleep between calls (<=4 req/s). NO burst concurrency.
- Timeout: >=30s (cold first call measured 6.5s).
- Partition scheme (Claude's discretion — recommended): `raw/{station}/{year}.ndjson`, sorted by date within a partition.
- Persist ONLY the 10 fields `normalizeObservations` yields: station,date,doy,t,tx,tn,f,fx,fg,dv,r. (doy is derived-but-persisted; date is the key.)
