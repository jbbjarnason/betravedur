# Phase 4 ‚Äî UI Design Contract

> Visual and interaction contract for the **Selection & Instant Recompute** phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **This spec EXTENDS the Phase 3 design system** (`site/src/styles/tokens.css`, `markers.css`, and `03-UI-SPEC.md`) ‚Äî it does not introduce a new one. All tokens, type roles, and colors below are the Phase 3 tokens reused; only the *new controls* (bottom control bar, scrubber, width buttons, year dropdowns, N-label surface, URL state) are specified here.

**Phase scope (contract is for THIS phase only):** the bottom control bar ‚Äî a day-of-year **scrubber** (window anchor) + segmented **width buttons** (`1 vika / 2 vikur / 3 vikur / 1 m√°nu√∞ur`); the **year-range selector** (`Fr√° [√°r]` / `Til [√°r]` dropdowns, bounded by the data); the honest **`me√∞altal N √°ra`** label surface (per-marker + global readout) with its `√≥fulln√¶gjandi g√∂gn` state; **instant client-side recompute/recolor** on any change (no fetch, no reload); and **URL-state encoding** (period + years + station + viewport).

**Explicitly out of scope (seams noted, not specified here):**
- Score color palette / legend / ranked "best stations" list ‚Äî **Phase 5**. Markers here re-render in the *existing Phase 3 neutral white-pill style* with the new averages; "recolor" = re-render with new numbers, NOT the score scale.
- Station-click chart panel ‚Äî **Phase 6** (marker click seam already stubbed in Phase 3; no handler added here).
- Mobile bottom-sheet polish, info "s√∂gulegt, ekki sp√°" panel, full loading / empty / no-data states ‚Äî **Phase 7**. This phase ships only the *functional* narrow-screen collapse (compact date stepper) and reuses the existing `√≥fulln√¶gjandi g√∂gn` marker state; it does NOT add new empty/loading chrome.

## Scrubber Anatomy

The window's **anchor** picker: a day-of-year slider operating in calendar-date / time-of-year terms (NOT a specific year). Feb 29 folded per the Phase 1 leap contract; the anchor is a day-of-year in 1..365.

Implemented as a native `<input type="range" min="1" max="365" step="1">` (built-in keyboard + a11y), restyled to the tokens. The *window* is `[anchor, anchor + widthDays ‚àí 1]` folded/wrapped via the domain `expandWindow` / `groupBySeasonYear` contracts; **wrapping windows (e.g. late Dec ‚Üí early Jan) are allowed** and render the same.

| Sub-element | Spec |
|-------------|------|
| Track | Full-width horizontal bar, height 4px, radius 2px. Unfilled = `--dominant`; the **selected-window span** (anchor ‚Üí anchor+width) is drawn as a filled `--ink` region on the track (so the user sees the *width*, not just a point). On a wrapping window the fill wraps from the right edge back to the left. |
| Current-anchor indicator (thumb) | A `--secondary` (white) circle, 16px visual / 44px hit area, 1px `--hairline` border + subtle shadow (matches marker-pill shadow language). Ink, never accent-red. Draggable + keyboard-movable. |
| Date readout | Directly above the thumb (or in the bar's readout slot on narrow widths): the anchor date in Icelandic, 13px Semibold, e.g. `20. j√∫l√≠` ‚Üí shows the window as `20.‚Äì26. j√∫l√≠` when it fits; else just the anchor date. Updates live during drag. |
| Month tick labels | 12 lowercase 3-letter Icelandic month abbreviations (`jan feb mar apr ma√≠ j√∫n j√∫l √°g√∫ sep okt n√≥v des`) at 11px muted, positioned at each month's start-of-month day-of-year. This is the tick **density**: month-level, not day-level (365 day ticks would be unreadable at this width). Optional finer un-labelled hairline ticks at week boundaries are allowed but must not clutter. |
| Keyboard operability | Native range gives: `‚Üê/‚Üí` = ¬±1 day, `Home` = doy 1 (`1. jan`), `End` = doy 365 (`31. des`), `PageUp/PageDown` = ¬±7 days (one week). Document the ¬±7 page step in code. The thumb is tab-focusable and shows the `--ink` focus outline. |
| Recompute trigger | Dragging fires continuous recompute ‚Äî **debounced** (Claude's discretion, target ~120ms trailing) so the map recolors smoothly without thrashing; a keyboard step recomputes immediately on `change`. Recompute is client-side only (see Instant Recompute). URL updated via `replaceState` on each settle (continuous refinement). |

## Width Segmented Buttons

A segmented control choosing the window length. Exactly **four** options, Icelandic labels, mutually exclusive (radio semantics).

| State | Spec |
|-------|------|
| Options (in order) | `1 vika` (7 days) ¬∑ `2 vikur` (14 days) ¬∑ `3 vikur` (21 days) ¬∑ `1 m√°nu√∞ur` (~30 days) |
| Markup | A `role="group"` (or a fieldset of `role="radio"` buttons) of four native `<button>`s sharing one rounded container with `--hairline` separators; radius 8px (matches marker/pill radius). Text labels (not icons) ‚Äî Claude's-discretion call resolved to **text**, because the Icelandic words are short and unambiguous and icons for "1 vs 2 vs 3 weeks" would be arbitrary. |
| Default (inactive) | `--secondary` fill, `--muted-ink` text, weight 400. |
| Active (selected) | `--ink` text, weight **600**, subtle `--dominant` inset fill (or `--secondary` with an ink 2px underline/inset) to read as pressed. `aria-pressed="true"` / `aria-checked="true"` on the active one; the others false. **Never uses accent red.** |
| Hit target | Each button ‚â• 44px tall, ‚â• 44px wide (padded). |
| Behavior | Selecting a width recomputes immediately (discrete change ‚Üí `pushState` is unnecessary for width; use `replaceState` ‚Äî width change is a refinement, not a navigation) and updates every marker's average. |
| Width‚Üídays mapping | `1 vika`=7, `2 vikur`=14, `3 vikur`=21, `1 m√°nu√∞ur`=30 (or 28; document the exact day count in code ‚Äî a fixed 30 is acceptable and simplest). The domain `WindowSpec` is `{ startDoy: anchor, endDoy: anchor + widthDays ‚àí 1 }` folded to 1..365. |

## Year-Range Selector ‚Äî `Fr√°` / `Til` Dropdowns

Two native `<select>`s picking the baseline year range averages are computed over. **Bounds derived from the committed data, never hardcoded.**

| Property | Value |
|----------|-------|
| Placement | Right end of the control bar (desktop), each `<select>` preceded by its label: `Fr√° [ 2016 ‚ñæ ]  Til [ 2026 ‚ñæ ]`. Labels 14px `--muted-ink`; `<select>` text 14px `--ink`. |
| Markup | `<label for="year-from">Fr√°</label><select id="year-from">‚Ä¶</select>` and likewise `Til`. Native `<select>` gives keyboard + screen-reader + platform-consistent behavior for free ‚Äî do NOT build a custom dropdown. |
| Option bounds | Populated at runtime from the **manifest**: `minYear = min(entry.from)` and `maxYear = max(entry.to)` across `manifest.stations` (the per-station high-water marks ‚Äî the manifest carries `{from, to}` per station, e.g. `{from:1949, to:2026}`). `Fr√°` options = `minYear..maxYear`; `Til` options = `minYear..maxYear`. | Guard `start ‚â§ end` | On change, if `Fr√° > Til`, clamp the other so the range stays valid (e.g. picking a `Fr√°` later than the current `Til` bumps `Til` up to match, or vice-versa). Never allow an inverted/empty range. |
| Behavior | Changing either dropdown recomputes every marker immediately (no fetch), and updates the `me√∞altal N √°ra` labels. Year change is a refinement ‚Üí `replaceState`. |
| Style | `--secondary` background, 1px `--hairline` border, radius 8px, 44px min height, `--ink` text. No accent. |

## `me√∞altal N √°ra` ‚Äî Honest Coverage Label Surface

The trust feature (SEL-03). `N` = **effective N from the Phase 1 honest-N contract (‚â•80% coverage, N‚â•3 qualifying years)** ‚Äî the *actual qualifying-year coverage*, NOT the picker span. This is `MarkerDatum.n` / `MarkerDatum.sufficient`, already computed by `computeMarkerDatum`. Two surfaces:

| Surface | Spec |
|---------|------|
| Per-marker | Each sufficient marker's callout carries its own coverage. Because the Phase 3 pill is compact, the full `me√∞altal N √°ra` string need not always be inline; the **acceptance-critical** requirement is that the value is *readable per station*. Minimum: render `me√∞altal {n} √°ra` as text associated with the marker (inline in the pill on wide layouts, or via the pill's `aria-label` / title + the global readout on tight layouts). The N MUST reflect that specific station's qualifying years, not the picker span. |
| Global readout | A single always-visible line in the control bar, 11px `--muted-ink`, reading the current selection's typical coverage ‚Äî format `me√∞altal N √°ra` (using a representative/median N, or a range `me√∞altal N‚ÄìM √°ra` if station coverage varies). This gives the user an at-a-glance honesty signal even before reading individual markers. |
| Insufficient state | When a station's `sufficient === false` (N < 3): its marker renders the **existing Phase 3 muted `√≥fulln√¶gjandi g√∂gn` pill** (`--marker-empty` tokens) ‚Äî no new state invented, no temp/wind/precip shown. The global readout, if the *whole* selection is thin (rare), likewise degrades to `√≥fulln√¶gjandi g√∂gn`. |
| Format | Icelandic, literally `me√∞altal N √°ra` (lowercase, `N` an integer, `√°ra` genitive plural ‚Äî correct for all N‚â•2; N is always ‚â•3 when shown so no `√°rs`/`√°ri` singular-agreement edge case arises). |

## Instant Recompute (no network)

The core loop. Changing the scrubber anchor, the width, or either year dropdown MUST recompute and re-render every marker **entirely client-side** ‚Äî no `fetch`, no reload, no navigation.

| Aspect | Contract |
|--------|----------|
| Compute path | Reuse `computeMarkerDatum(meta, derivedFile, window)` (already parameterized by `WindowSpec` and coverage-gated). The derived files are already loaded in memory from Phase 3's boot; recompute iterates them with the new `WindowSpec` + year-range filter ‚Äî no new fetch. |
| Year-range filter | The year range constrains which season-years feed `qualifyingYears`/`effectiveN`. Passed alongside the window into the recompute (the domain math already qualifies per season-year; the picker just bounds the candidate set). |
| Re-render | Reuse Phase 3's **idempotent** `attachCompositeRenderer` (WR-04 fix) and `installMarkerLayer` ‚Äî update the layer's data / re-render the composite in place. **Do NOT stack listeners or mount a fresh overlay per change** (the idempotency guarantee is why this is safe). |
| Recolor semantics | "Recolor" this phase = re-render pills with the *new numbers* in the **existing neutral white-pill style**. The score color scale is Phase 5 ‚Äî do not introduce it. |
| State single-source | A single selection-state object `{ anchorDoy, widthDays, fromYear, toYear, stationId?, viewport }` is the one source of truth: URL ‚áÑ state ‚áÑ recompute. Phase 5/6 read from it. Encapsulate so those phases extend, not rewrite. |
| Performance | Recompute is measurably instant; the E2E asserts **zero network requests fire** on a selection change (see acceptance criteria). Scrubber-drag recompute is debounced (~120ms) to stay smooth. |

## URL State (UX-02)

The full selection is encoded in URL query params so a copied link restores the exact view ‚Äî the static-site substitute for accounts/saved state.

| Aspect | Contract |
|--------|----------|
| Encoded params | period **anchor day-of-year** + **width**, year range **from**/**til**, **selected station id** (when one is selected ‚Äî the Phase 6 seam; may be absent this phase), and **map viewport** lat/lng/zoom. Exact param names are Claude's discretion (suggested: `d` or `doy`, `w`, `fra`, `til`, `st`, `lat`, `lng`, `z`) ‚Äî document them in code. |
| History strategy | **`replaceState`** for continuous refinements (scrubber drag, width change, year change, pan/zoom) so the back button isn't spammed; **`pushState`** for discrete navigations (station select, once wired) ‚Äî per FEATURES.md guidance. |
| Restore | On load, if params are present, hydrate the selection-state from them (clamped to valid bounds ‚Äî an out-of-range year in a stale link snaps to the nearest valid bound, never errors). If absent, apply the **default selection** below. |
| Default selection (no params) | window = **current week** (anchor = today's day-of-year at load, width = `1 vika`) over the **last 10 available years** (`toYear = maxYear`, `fromYear = max(minYear, maxYear ‚àí 9)`). Today is NOT hardcoded ‚Äî the anchor derives from the current date at load; the year range derives from the manifest's latest 10 years. This replaces the fixed Phase 3 `DEFAULT_WINDOW` constant. |
| Viewport sync | Pan/zoom writes lat/lng/zoom via `replaceState` (debounced on `moveend`); reloading the link restores the same framing within the Phase 3 min/max zoom + maxBounds. |

## Narrow-Screen Collapse (functional only ‚Äî full mobile polish is Phase 7)

On narrow widths the horizontal scrubber has too little room for readable ticks and a draggable thumb, so it **collapses to a compact date stepper** ‚Äî a *functional* fallback, NOT the polished Phase 7 bottom sheet.

| Breakpoint | Behavior |
|------------|----------|
| ‚â• ~640px | Full bar: scrubber + width buttons + year dropdowns in one row (may wrap width buttons under the scrubber at mid widths). |
| < ~640px | Scrubber replaced by a **compact date stepper**: `[ ‚Äπ ] 20. j√∫l√≠ [ ‚Ä∫ ]` ‚Äî two 44px `‚Äπ`/`‚Ä∫` buttons stepping the anchor ¬±1 day (with a ¬±7 accelerator on hold/long-press optional), flanking the live Icelandic date readout. Width buttons remain (may shrink to `1v / 2v / 3v / 1m` shorthand ONLY if the full words don't fit ‚Äî prefer full words). Year dropdowns remain (native `<select>` is already mobile-friendly). |
| Bar height (narrow) | May grow to two stacked rows; keep total ‚â§ ~120px so the map stays usable. Full bottom-sheet drag/expand is deferred to Phase 7 ‚Äî do not build it here. |

The stepper drives the identical selection-state + recompute + URL path as the scrubber; it is purely an input-affordance swap.

## Accessibility Notes

- **Scrubber (slider):** native `<input type="range">` provides `role="slider"`, `aria-valuemin/max/now` automatically. Add `aria-label="Velja t√≠mabil"` (or `aria-labelledby` the date readout) and keep the live date readout in the DOM (`aria-live="polite"` on the readout) so screen-reader users hear the anchor date as it changes. Keyboard: `‚Üê/‚Üí` ¬±1 day, `PageUp/Down` ¬±7, `Home`/`End` = year ends (documented above).
- **Width buttons:** grouped with `role="group"` + a group label (`aria-label="Lengd t√≠mabils"`), each button `aria-pressed`/`aria-checked` reflecting active state, keyboard-operable (native buttons ‚Äî Enter/Space), arrow-key roving optional.
- **Year dropdowns:** each `<select>` has an associated visible `<label>` (`Fr√°`, `Til`) via `for`/`id`. Native keyboard + screen-reader support; no custom ARIA needed.
- **Focus:** every control shows a visible **`2px solid var(--ink)`, offset 2px** focus outline ‚Äî the same skeleton Phase 3 declared for markers. Never rely on accent-red for focus.
- **Contrast (reuse Phase 3 verified values):** `--ink` on white ‚âà 13:1 (AAA); `--muted-ink` on white ‚âà 5.9:1 (AA) ‚Äî used only for ‚â•11px tick/label text; the bar's `rgba(255,255,255,0.92)` + blur keeps text at AA over dark map regions beneath (verify over ocean/dark land, same as the Phase 3 header check).
- **Reduced motion:** any recompute/recolor transition and any scrubber-fill animation must respect `prefers-reduced-motion` (instant swap, no easing) ‚Äî continues the Phase 3 marker rule.
- **Color not sole channel:** active width = fill + weight + `aria-pressed`; anchor = shaped thumb + text date; insufficient = `√≥fulln√¶gjandi g√∂gn` text. No datum or state is conveyed by hue alone.
- **Tab order:** scrubber/stepper ‚Üí width buttons ‚Üí `Fr√°` ‚Üí `Til`, left-to-right matching visual order.

## Acceptance-Checkable Visual Criteria (Playwright UI review)

Objective assertions the automated E2E can make against the running preview build:
