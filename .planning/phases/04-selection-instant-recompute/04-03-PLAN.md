Make the full selection shareable and restorable (UX-02): serialize the store to URL query params on every change (`replaceState` for continuous refinements, `pushState` for discrete), read the URL only at boot and on `popstate` (the loop-proof asymmetry), and apply a data-derived default selection when no params are present (SEL-02 bounds from the manifest union). Close the phase with E2E: crafted-URL → exact-view restore, default-when-no-params, and back-button popstate.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL query params → store (paramsToState) | URL params are attacker-controllable input (ASVS V5). Every field must be validated/clamped before it reaches the store or the domain math. |
| store → map viewport | store→map jumpTo only on boot/popstate; map→store only on moveend — the asymmetry prevents a camera loop. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | Tampering / DoS | paramsToState (malformed URL) | mitigate | Clamp doy∈[1,365], w∈{7,14,21,30}, fra/til∈bounds & fra≤til, viewport∈maxBounds, zoom∈[4,12]; coerce non-numeric → fallback; never throw, never NaN (Task 1 garbage-input test). |
| T-04-06 | Injection (XSS) | station id / any URL string reflected into DOM | mitigate | `st` parsed to an integer only; NO URL-derived string is innerHTML'd — labels come from trusted stations.json, not the URL (RESEARCH Security Domain). |
| T-04-07 | DoS (local) | history flooding on continuous input | mitigate | replaceState for scrubber-drag + pan/zoom collapses continuous moves to one history entry (Pattern 3). |
| T-04-SC | Tampering | npm installs | mitigate | Zero new packages (URLSearchParams/History/Intl are platform APIs). No supply-chain surface. |
</threat_model>
