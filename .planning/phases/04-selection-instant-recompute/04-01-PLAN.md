<objective>
Build the selection-state single source of truth and the instant client-side recompute path — the load-bearing core the whole phase (and Phase 5/6) rests on. Add the one domain signature change (a `yearRange` param on `computeMarkerDatum`, SEL-02/03), a ~40-line observable store, the pure `anchorToWindow` converter (SEL-01), and a boot-time station→DerivedFile cache so recompute runs over already-loaded data with NO network fetch (SEL-04). Expose `window.__store` so downstream E2E can drive selection deterministically.

Purpose: This is the thinnest working vertical slice — after it, driving `window.__store.set(...)` recomputes and re-renders every marker with new averages and zero fetches, provably. Plans 02 (selector UI) and 03 (URL state) both write to this store.

Output: `store.ts`, `window.ts`, `recompute.ts` (+ their tests), an extended `computeMarkerDatum`/`averages.test.ts`, and a rewired `main.ts` that caches files at boot and recomputes on store change.
</objective>
