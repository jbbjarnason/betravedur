<objective>
Build the bottom control bar and wire every control to the Plan 01 store so the visitor can drive selection: a native day-of-year scrubber (SEL-01 anchor), four width buttons (SEL-01 length), Frá/Til year dropdowns bounded by the data (SEL-02), and an always-visible `meðaltal N ára` global readout (SEL-03). Each control writes to the store; the Plan 01 debounced subscriber recomputes and re-renders markers with no fetch (SEL-04). Prove the loop end-to-end with Playwright: controls render, and a selection change updates a marker with zero network requests.

Purpose: This is the second vertical slice — after Plan 01 made "state change → recompute" true, this makes the *user* able to change state through real controls. It reuses Phase 3 tokens/markers verbatim; no new design system, no accent-red on controls, no score palette.

Output: `controls.css`, `scrubber.ts`, `widthButtons.ts`, `yearRange.ts`, `controlBar.ts` (+ a scrubber unit test for the Icelandic date label), main.ts wiring, and `selection.spec.ts` E2E.
</objective>
