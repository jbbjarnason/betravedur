## Phase Goal

**As a** visitor planning an Icelandic trip, **I want to** see stations colored by their combined weather score with a legend, a ranked best-stations list, and a plain explainer, **so that** I can tell at a glance where the weather has historically been best and trust how that score was reached.

> Note (MVP_MODE): the ROADMAP `**Goal:**` line is outcome-shaped but not in `As a / I want to / so that` form. This story is derived faithfully from that goal's explicit role + capability + outcome. If a stricter source story is required, run `/gsd mvp-phase 5` to set it in ROADMAP.md.

<objective>
Surface the actual answer to the user's question: a collapsible right-docked "Bestu staðir" panel ranking scored stations (desc) for the current selection, excluding ófullnægjandi-gögn stations and badging án-úrkomu ones. A row click flies the map to that station via `easeTo`, sets `stationId`, and writes the URL `st` param — reusing the Phase-4 seams, no chart panel (Phase-6 seam kept clean).

Purpose: SCORE-02 (ranked list) and the row-click fly-to/select. The list re-renders on the same debounced recompute hook as the markers (no fetch, no viewport-churn), and the fly-to is a stationId-only subscriber guarded by `viewportMatches` — no new camera↔store loop.

Output: `rankedList.ts` (+ pure `rankStations` + tests), panel CSS, `main.ts` wiring (refresh hook + stationId→easeTo subscriber), and score.spec criteria 5-10, 12-13.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Tampering (XSS) | ranked-row station name | mitigate | Render name via `element.textContent` / `setAttribute`, NEVER innerHTML (RESEARCH Security Domain V5) — the name originates from a committed data file but is treated untrusted-by-default |
| T-05-06 | Denial of Service | rankStations on score:null | mitigate | Filter score===null BEFORE sort so the comparator never dereferences null; combine() guarantees score is number|null |
| T-05-07 | Availability | easeTo → moveend re-loop | mitigate | Drive easeTo from a stationId-only subscriber; guard the outbound viewport write with viewportMatches (Phase-4 Pitfall 4 discipline) — no isUpdating flag |
| T-05-SC | Tampering | npm installs | accept | Zero new deps (native <ol>/<button>, existing maplibre) |
</threat_model>
