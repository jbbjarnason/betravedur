# Phase 5 — UI Design Contract

> Visual and interaction contract for the **Score Coloring & Ranking** phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **This spec EXTENDS the Phase 3 + Phase 4 design system** (`site/src/styles/tokens.css`, `markers.css`, `controls.css`, `03-UI-SPEC.md`, `04-UI-SPEC.md`) — it does NOT introduce a new one. All spacing, type roles, and existing color roles below are the Phase 3/4 tokens reused verbatim. The ONLY new tokens introduced are the **sequential score-color ramp** (`--score-0` … `--score-10`) and the panel/legend chrome that consume them. Accent red (`--accent` `#C0392B`) stays reserved to the temperature numeral.

**Phase scope (contract is for THIS phase only):**
- **Score-colored markers** — the existing white pill gains a **score-ramp ring/border** plus a **numeric score badge**; pill body stays white, temperature numeral stays red.
- **Legend** — a floating corner panel with the color scale (score labels) AND a folded-in expandable **"hvernig er einkunnin reiknuð?"** explainer (weights: úrkoma 40% / vindur 30% / hiti 30% + "án úrkomu" renormalization).
- **Ranked "Bestu staðir" panel** — a collapsible right-side list of stations ranked by score (desc) for the current selection: rank · name · score · "án úrkomu" badge; click a row → fly-to/highlight that marker + update the URL `st` param.
- Everything recomputes through the **existing Phase 4 store/recompute path** — NO new network fetch.

**Explicitly out of scope (seams noted, not specified here):**
- **Station chart panel — Phase 6.** The marker/row click in THIS phase only flies-to + highlights + sets `st`; it opens **no panel**. Keep the click handler seam clean so Phase 6 can attach the panel to the same `st`-select event.
- **Mobile bottom-sheet, info "sögulegt, ekki spá" panel, full loading/empty/no-data polish — Phase 7.** Here the list + legend degrade to a **functional toggle** on narrow screens only; no polished sheet chrome.
- **Reverse "worst weather" ranking (RANK-04), adjustable weight sliders (WGT-01)** — deferred (v1.x / v2). Do not build.

## Color

**Existing Phase 3/4 color roles reused verbatim.** The ONLY new tokens are the score ramp. Accent red (`--accent`) **remains reserved to the temperature numeral** — the score ramp is a chromatically distinct blue→teal→green family and **must not** reuse or approach the accent red.

### New tokens — sequential score-color ramp

A **colorblind-safe SEQUENTIAL ramp** keyed to the 0–10 score. Low = muted cool slate; high = vivid green ("gott veður = grænt"). It is **NOT** a red–green diverging scale (accent red is reserved AND red–green is colorblind-hostile). The ramp is **monotonic in luminance** (dark→light as score rises), so it survives grayscale — a built-in backup for the color-not-sole-channel rule — and moves along a blue→teal→green hue axis that protanopia/deuteranopia/tritanopia all preserve.

| Token | Score | Hex | Notes |
|-------|-------|-----|-------|
| `--score-0` | 0 | `#5B6B7A` | muted slate (low, cool, desaturated) |
| `--score-1` | 1 | `#516F82` | |
| `--score-2` | 2 | `#457489` | |
| `--score-3` | 3 | `#347B8B` | |
| `--score-4` | 4 | `#1F8389` | deep teal |
| `--score-5` | 5 | `#0E8A82` | |
| `--score-6` | 6 | `#159079` | |
| `--score-7` | 7 | `#33966A` | |
| `--score-8` | 8 | `#4E9C58` | green |
| `--score-9` | 9 | `#6BA23F` | |
| `--score-10` | 10 | `#84A81F` | vivid yellow-green (highest / most vivid — "grænt") |

- **Colorblind-safety (verified):** the ramp is strictly increasing in WCAG relative luminance (0.141 → 0.330), so under any dichromatic simulation the ordering is still readable as a light/dark gradient. Hue never crosses the red–green confusion axis (no reds/magentas), so protanopes/deuteranopes read it. Accent red `#C0392B` (relLum 0.143) is hue-distinct (warm red vs cool blue–green) from every ramp stop — a marker's red temp numeral can never be confused with its score ring.
- **Score binning:** the continuous 0–10 score maps to the ramp by rounding to the nearest integer stop for the discrete legend swatches; the **marker ring** may either snap to the nearest stop or interpolate — Claude's discretion, but the legend and marker must agree at the labeled stops.

### How the ramp applies (marker)

The neutral Phase 3/4 white pill becomes score-colored **on two channels, never overwriting the white body or the red temperature**:

| Channel | Spec |
|---------|------|
| **Score ring / border** | The pill's 1px `--hairline` border is replaced by a **2px ring in the station's score-ramp color** (`--score-{n}`). The white body, red temp, ink wind, and shadow are unchanged. The ring reads against both the white pill (contrast ≥2.76:1 for every stop) and the muted `--dominant` basemap (≥2.31:1) — perceptible as an essential boundary because it is paired with the numeric badge (never the sole channel). |
| **Score badge** | A small chip appended to the pill (leading edge, before the temperature, or trailing — Claude's discretion; must not displace the temp/wind/precip order's readability). The badge shows the **numeric score** (`7,8`) at Compact-value 13px Semibold, **ink `#1F2933` on a white chip** (13:1 — legibility never depends on the ramp), with a 2px `--score-{n}` ring so it doubles as a color key. **The numeral is the required non-color channel; the ring color is the redundant channel.** |

> **Why the numeral is on white, not on the ramp fill:** a filled-ramp badge would force a per-stop text-color crossover (white text legible only for scores 0–6, ink text only for 7–10, with score 6 marginal either way). Keeping the numeral ink-on-white makes score legibility ramp-independent and AA-safe at every stop. The ramp does its job as the **ring** channel on both the pill and the badge.

### `ófullnægjandi gögn` and `án úrkomu` on the ramp

| Case | Datum condition | Marker treatment |
|------|-----------------|------------------|
| **Insufficient (`score:null`, unscorable)** | `sufficient === false` OR `score === null` (domain: `contributing` empty) | **Existing Phase 3 muted pill** (`--marker-empty` tokens) — **NO score ring, NO score badge, OFF the ramp entirely.** Renders `ófullnægjandi gögn`. **Excluded from the ranked list.** |
| **Scored "án úrkomu"** | `score !== null` AND `missingRain === true` (temp+wind renormalized, rain absent) | **Fully colored + badged + ranked** on the renormalized score. Carries the existing "án úrkomu" signal (precip glyph omitted per Phase 3) AND an **"án úrkomu" badge** in the ranked row. It is NOT muted — a rain-less station is scored fairly, not penalized. |

> Domain note for the planner: `missingRain === true` does **not** imply `score === null`. A station can be `missingRain: true` **and** scored (án úrkomu) — colored + ranked. Only an **empty `contributing`** (⇒ `score === null`) is the off-scale/excluded case. Extend `MarkerDatum` (currently lacking score fields) to carry `score`, `contributing`/`missingRain` from `combine()`; do NOT recompute the math.

### Existing roles reused

| Role | Value (existing token) | Usage in this phase's panels |
|------|------------------------|------------------------------|
| Dominant (60%) | `--dominant` `#E8EBED` | Ranked-row hover/selected fill; legend track backdrop |
| Secondary (30%) | `--secondary` `#FFFFFF` | Legend + ranked-panel surface at `rgba(255,255,255,0.92)` + `backdrop-filter: blur(8px)` — **identical treatment to the Phase 3 header + Phase 4 control bar** so all chrome reads as one design language |
| Ink | `--ink` `#1F2933` | Panel titles, station names, score numerals (badge + row), focus outline (`2px solid var(--ink)`, offset 2px) |
| Muted ink | `--muted-ink` `#5B6670` | Rank numerals, "án úrkomu" badge, empty-state line, legend endpoint captions |
| Marker empty | `--marker-empty-*` | Reused unchanged for the `ófullnægjandi gögn` off-scale pill |
| Hairline | `--hairline` `rgba(31,41,51,0.10)` | Panel borders, row separators, legend swatch borders, collapse-toggle border |

**Accent reserved for:** the temperature numeral inside a marker, and nothing else (unchanged). The score ramp is a separate token family and never the accent.

**Destructive:** none — this phase has no destructive actions. Selecting a station (row click) is non-destructive, instantly reversible, and URL-encoded (`st`). No confirmations.

**Color is never the sole channel:** every score is accompanied by its **numeral** (badge on the pill, value in the row); the ramp is monotonic in luminance (grayscale-legible); `án úrkomu` carries a **text badge** + the omitted precip glyph; `ófullnægjandi gögn` carries **text** + the muted tone + exclusion from the list. No datum or state is conveyed by hue alone.

## Layout — Legend & Ranked Panel

Two new floating surfaces dock into regions Phase 3 reserved ("a right-edge column is reserved conceptually for the Phase 5 ranked list"), without fighting the 56px header (top) or the 72–96px control bar (bottom).

| Surface | Placement |
|---------|-----------|
| **Ranked "Bestu staðir" panel** | `position: fixed; right: 0; z-index: 10;` docked to the **right edge**, top offset `--space-lg` (24px) below the header, bottom offset clearing the control bar (`bottom: calc(var(--bar-height, 100px) + var(--space-lg))` — reuse the `--bar-height` var the control bar already writes on mount/resize). Width ~260–300px on desktop. Surface = header/control-bar treatment (`rgba(255,255,255,0.92)` + blur + 1px `--hairline`, radius 8px). **Collapsible** (see below) so it never permanently crowds the map. |
| **Legend panel** | `position: fixed; z-index: 10;` docked to a **corner that fights neither the control bar nor the ranked panel** — **bottom-left** on desktop (the control bar's global readout is left-of-center but short; the legend sits above the bar with a `--space-lg` gap, `left: --space-md`). Same surface treatment. Compact: the color scale + the folded explainer. Must not exceed ~320px wide or occlude the central station band at zoom 6. |

**Occlusion budget (continues Phase 4):** header (56px) + control bar (≤96px) + the right panel (≤300px wide) + the bottom-left legend must leave the **central map band** (bulk of Iceland's stations at zoom 6) visible. The right panel is collapsible precisely to honor this; collapsed, it is a slim tab.

**Z-order:** all new panels share the header/control-bar tier (`z-index: 10`), above markers (z5) and canvas (z0). They must not overlap each other at rest on desktop (right panel right-docked, legend bottom-left).

## Legend Anatomy

One place for all score meaning (the transparency differentiator — WeatherSpark's opaque Tourism Score is the anti-pattern to beat). The color scale AND the explainer live together.

```
 ┌───────────────────────────────┐
 │ Einkunn                        │  ← title (14px 600)
 │ ▉▉▉▉▉▉▉▉▉▉▉  betra →            │  ← continuous ramp bar, low→high
 │ 0    2    4    6    8    10     │  ← score labels (11px muted, tabular)
 │ verra ←            → betra      │  ← endpoint captions (11px muted)
 │ ───────────────────────────    │
 │ ▸ hvernig er einkunnin reiknuð?│  ← <summary> disclosure (14px 600)
 │   (expands ↓)                  │
 └───────────────────────────────┘
```

| Sub-element | Spec |
|-------------|------|
| Title | `Einkunn`, Section-title 14px Semibold ink. |
| Color scale | A horizontal ramp bar rendering `--score-0` → `--score-10` (CSS `linear-gradient` across the 11 stops, OR 11 discrete swatches). Height ~10–12px, radius 4px, 1px `--hairline` border so the low (dark slate) end reads against the panel. |
| Score labels | Tick labels `0 2 4 6 8 10` (11px `--muted-ink`, tabular) beneath the ramp, aligned to their stops. |
| Endpoint captions | `verra` at the low end, `betra` at the high end (11px muted) — plain-Icelandic reinforcement that up = better, so the scale is legible even monochrome. |
| Explainer disclosure | A native `<details><summary>hvernig er einkunnin reiknuð?</summary>…</details>` (14px 600 summary, chevron rotates on open). Built-in keyboard (Enter/Space toggles) + `aria-expanded` for free. Collapsed by default. |
| Explainer body (expanded) | Plain-Icelandic prose (14px 400), stating the weights and the renormalization: **`úrkoma 40%`**, **`vindur 30%`**, **`hiti 30%`**, then the án-úrkomu rule. See Copywriting Contract for the exact strings. |

## Ranked "Bestu staðir" Panel

The **actual answer to the user's question** (FEATURES.md) — make it prominent, not buried. Lists stations ranked by score (descending) for the **current selection**, updating live on every selection change through the same recompute path.

```
 ┌──────────────────────────────┐
 │ Bestu staðir            [ ⌄ ] │  ← title + collapse toggle
 ├──────────────────────────────┤
 │ 1.  Egilsstaðir      8,4      │  ← rank · name · score
 │ 2.  Akureyri  án úrkomu 8,1   │  ← án úrkomu badge before score
 │ 3.  Höfn             7,9      │
 │ …                            │
 └──────────────────────────────┘
```

| Element | Spec |
|---------|------|
| Container | `<section aria-label="Bestu staðir">` with a `<header>` (title + collapse toggle) and an `<ol>` of rows. Ordered-list semantics carry the ranking to assistive tech; the visible `1. 2. 3.` numerals are redundant-but-explicit. |
| Title | `Bestu staðir`, Section-title 14px Semibold ink. |
| Row anatomy | Each `<li>` is a single `<button>`/row (whole row is the 44px-min click target): **rank** (`1.` 11px muted, tabular) · **station name** (14px 400 ink, truncates with ellipsis if long) · optional **`án úrkomu` badge** (11px muted, a small `--hairline`-bordered chip) · **score** (`8,4` 13px Semibold ink, right-aligned, tabular). A tiny score-ramp swatch/dot (`--score-{n}`) may prefix or accompany the score as the redundant color channel — score numeral is required, color is optional reinforcement. |
| Row density | Compact: `--space-sm` (8px) vertical padding, `--space-xs`–`--space-sm` gaps. Comfortable enough for a 44px hit area. |
| Ordering | Strictly by `score` descending. Ties: stable secondary sort (Claude's discretion — e.g. by name or station id) so order doesn't flicker on recompute. |
| Exclusions | Stations with `score === null` (`ófullnægjandi gögn` / unscorable) are **NOT listed**. Only scored stations (including `án úrkomu`) appear. |
| Length | Show the ranked stations for the current selection. If the list is long, it scrolls within the panel's fixed height (overflow-y auto); the panel does not grow past its bottom bound. A "top N" cap is Claude's discretion but v1 may simply list all scored stations, scrollable. |
| Live update | Subscribes to the same `latestData` the markers subscribe to (Phase 4 store). Every scrubber/width/year change re-sorts + re-renders the rows with the new scores — **no fetch** (same guarantee as markers). |

### Collapsed vs expanded (desktop)

| State | Behavior |
|-------|----------|
| Expanded (default desktop) | Full panel with title + scrollable ranked list, right-docked. |
| Collapsed | The panel collapses to a slim vertical **tab/handle** at the right edge (title `Bestu staðir` or an icon), 44px-min hit target, `aria-expanded="false"`. Clicking it re-expands. Collapse state may be persisted (Claude's discretion — not required in the URL). |
| Toggle | A native `<button>` in the panel header (chevron), `aria-expanded` reflecting state, `aria-controls` the list region. Keyboard-operable (Enter/Space). |

### Narrow-screen treatment (functional only — Phase 7 does the polished sheet)

| Breakpoint | Behavior |
|------------|----------|
| ≥ ~640px | Right-docked collapsible panel as above; legend bottom-left. |
| < ~640px | The right panel is **not** always-open (it would cover the map). It degrades to a **simple toggle**: a compact button (e.g. `Bestu staðir` chip, top-right under the header) that shows/hides the list as an overlay. The legend likewise collapses to a toggle chip. **This is a functional show/hide only — NOT a draggable bottom sheet** (Phase 7). Keep total chrome from occluding the map when toggled shut. |

### Click → fly-to / highlight (Phase 6 seam kept clean)

| Aspect | Contract |
|--------|----------|
| On row click | (1) **Fly the map** to the station's marker via MapLibre `easeTo` (reuse existing map + marker; center on the station, keep a sensible zoom); (2) **highlight** the target marker (a transient emphasis — e.g. a thicker ring or brief scale, respecting reduced motion); (3) **set the selected station** → write the URL **`st` param** via the Phase 4 store (`pushState` for a discrete navigation, per Phase 4's history strategy). |
| No panel this phase | The click does **NOT** open a chart panel. Phase 6 attaches its panel to the same `st`-select event/seam. Keep the handler a single "select station" action so Phase 6 extends, not rewrites. |
| Reduced motion | `easeTo` duration respects `prefers-reduced-motion` (instant jump / minimal easing) — continues the Phase 3/4 rule. |
| Reciprocal highlight | Selecting a station (via row OR, later, a marker click) should visibly mark BOTH the marker and — Claude's discretion — the matching ranked row (e.g. selected-row `--dominant` fill), so the two views stay in sync. |

### Empty state (no station qualifies)

When **no station has a score** for the current selection (all unscorable / `ófullnægjandi gögn` — rare, defensive), the panel body shows a plain line instead of an empty `<ol>` or a crash: heading + one-sentence body (see Copywriting). The panel itself still renders (title + message). This is the phase-appropriate empty state; the fuller loading/empty/no-data chrome is Phase 7.

## Recompute Integration (no network)

Continues the Phase 4 core loop. The score **coloring** (marker ring + badge), the **legend** (static — the ramp doesn't change with selection, but the "án úrkomu" prevalence does not affect it), and the **ranked list** all read from the existing store's `latestData`.

| Aspect | Contract |
|--------|----------|
| Compute path | The score is already computed by the domain `combine()` into the (extended) `MarkerDatum` on each recompute. This phase **reads** `score` / `missingRain` / `contributing` — it does NOT recompute the math and adds NO fetch. |
| Marker re-render | Extend the existing idempotent composite renderer (`markers.ts` `renderComposite`) to add the ring + badge from `datum.score`. Do NOT stack listeners or mount a fresh overlay — reuse the Phase 3/4 idempotency guarantee. `symbol-sort-key` may additionally key on score (higher score = higher collision priority) per the code-context hint. |
| List/legend subscribers | The ranked list re-sorts + re-renders on the same `latestData` update. Legend is static chrome (renders once; the explainer content is fixed copy). |
| Performance | A selection change fires **zero network requests** (same E2E guarantee as Phase 4). Marker recolor + list re-sort is client-side and instant; scrubber-drag stays debounced (existing ~120ms). |

## Accessibility Notes

- **Color not the sole channel (score):** every score carries a **numeral** (badge + row value) and the ramp is **monotonic in luminance** (grayscale-legible); `betra`/`verra` captions state the direction in words. The ring color is always redundant to the numeral.
- **Ramp contrast:** the score ring is an essential boundary paired with the numeric badge; it renders at ≥2.76:1 against the white pill and ≥2.31:1 against the `--dominant` basemap at every stop (verified). The **score numeral** is ink `#1F2933` on a white chip (13:1, AAA) — legibility is ramp-independent. Ranked-row text is ink-on-white (13:1); muted rank/badge text at 11px on white ≈ 5.9:1 (AA, unchanged Phase 3 value).
- **List semantics:** the ranked list is an `<ol>` (ordered) inside a labelled `<section aria-label="Bestu staðir">`; each row is a native `<button>` (or `role` equivalent) so it is keyboard-focusable and Enter/Space activates the fly-to/select. Visual `1. 2. 3.` numerals are redundant to the `<ol>` order.
- **Legend + explainer keyboard:** the explainer uses native `<details>`/`<summary>` (Enter/Space toggles, `aria-expanded` free); the collapse toggle is a native `<button>` with `aria-expanded` + `aria-controls`. The legend is a labelled region (`aria-label="Skýring á einkunn"`).
- **Focus:** every interactive element (rows, toggles, disclosure) shows the standard **`2px solid var(--ink)`, offset 2px** focus outline (the Phase 3/4 skeleton). Never accent-red.
- **Marker focus (Phase 6 seam active-ish):** the ranked-row click SELECTS a station and highlights its marker — author the marker highlight so a Phase 6 focus ring can reuse the same "selected" state without restructuring. Do not add the chart panel.
- **Reduced motion:** the `easeTo` fly-to and any marker-highlight transition respect `prefers-reduced-motion` (instant/minimal). Continues the Phase 3/4 rule.
- **Live region (optional):** when a row selection flies-to a station, an `aria-live="polite"` announcement of the selected station name is acceptable (Claude's discretion) so screen-reader users hear the result of the jump.
- **Tab order:** legend (scale → explainer) and ranked panel (collapse toggle → rows top-to-bottom) follow visual order; the new panels come after the Phase 4 controls in the document/tab flow, not before the map controls.

## Acceptance-Checkable Visual Criteria (Playwright UI review)

Objective assertions the automated E2E can make against the running preview build:

1. **Markers carry a score-derived color/badge:** at least one rendered marker has a score-ramp ring color (a `--score-*` value / non-`--hairline` border) AND a visible numeric score badge matching `/\d,\d/` (Icelandic comma), distinct from the temperature `°` value.
2. **Score recolors on selection change:** changing the scrubber/width/year updates at least one marker's score badge value (and/or ring color) **with zero network requests fired** during the interaction (`page.on('request')` count === 0).
3. **Legend renders with a color scale:** a legend region (`aria-label="Skýring á einkunn"` or the `Einkunn` title) is present, containing a rendered color-scale element and the endpoint captions `verra` and `betra`.
4. **Explainer expands showing weight text:** clicking/activating `hvernig er einkunnin reiknuð?` sets `aria-expanded="true"` and reveals text containing `úrkoma 40%`, `vindur 30%`, and `hiti 30%`.
5. **Ranked panel present + titled:** a `Bestu staðir` panel is rendered with an `<ol>` of rows.
6. **Ranked list rows in descending score order:** the rendered rows' numeric scores are monotonically non-increasing top-to-bottom (each row's parsed score ≤ the previous row's).
7. **Ranked list updates with selection:** after a selection change the ranked list's contents/order change (at least one row's score value differs), with **zero network requests** fired.
8. **`ófullnægjandi` station absent from list:** a station that renders the muted `ófullnægjandi gögn` pill for the current selection does NOT appear as a row in the `Bestu staðir` list.
9. **`án úrkomu` station is ranked + badged:** a station scored `missingRain` (án úrkomu) DOES appear in the ranked list, carrying the `án úrkomu` badge, and its marker is colored (has a score ring/badge), not muted.
10. **Clicking a row moves the map / selects the station:** activating a ranked row changes the map view (center/zoom changes via `easeTo`) AND sets the URL `st` param to that station — with **no chart panel opened** (Phase 6 seam: assert no panel element appears).
11. **No new fetch for coloring/ranking:** none of the above interactions fire a network request (the coloring + ranking are pure client-side reads of the existing store).
12. **Empty state:** for a selection where no station is scorable, the `Bestu staðir` panel shows the `Engin einkunn` message (not an empty list, no throw/white-screen).
13. **Collapse toggle:** the ranked panel's collapse `<button>` toggles `aria-expanded`, hiding/showing the list; collapsed, it leaves the central map band visible.
14. **Score number format:** every rendered score (badge + rows) matches the Icelandic one-decimal comma format `/^\d{1,2},\d$/` (e.g. `7,8`, `10,0`), never a `.`-separated value.
