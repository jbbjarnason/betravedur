# Phase 5 вҖ” UI Design Contract

> Visual and interaction contract for the **Score Coloring & Ranking** phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **This spec EXTENDS the Phase 3 + Phase 4 design system** (`site/src/styles/tokens.css`, `markers.css`, `controls.css`, `03-UI-SPEC.md`, `04-UI-SPEC.md`) вҖ” it does NOT introduce a new one. All spacing, type roles, and existing color roles below are the Phase 3/4 tokens reused verbatim. The ONLY new tokens introduced are the **sequential score-color ramp** (`--score-0` вҖҰ `--score-10`) and the panel/legend chrome that consume them. Accent red (`--accent` `#C0392B`) stays reserved to the temperature numeral.

**Phase scope (contract is for THIS phase only):**
- **Score-colored markers** вҖ” the existing white pill gains a **score-ramp ring/border** plus a **numeric score badge**; pill body stays white, temperature numeral stays red.
- **Legend** вҖ” a floating corner panel with the color scale (score labels) AND a folded-in expandable **"hvernig er einkunnin reiknuГ°?"** explainer (weights: Гәrkoma 40% / vindur 30% / hiti 30% + "ГЎn Гәrkomu" renormalization).
- **Ranked "Bestu staГ°ir" panel** вҖ” a collapsible right-side list of stations ranked by score (desc) for the current selection: rank В· name В· score В· "ГЎn Гәrkomu" badge; click a row вҶ’ fly-to/highlight that marker + update the URL `st` param.
- Everything recomputes through the **existing Phase 4 store/recompute path** вҖ” NO new network fetch.

**Explicitly out of scope (seams noted, not specified here):**
- **Station chart panel вҖ” Phase 6.** The marker/row click in THIS phase only flies-to + highlights + sets `st`; it opens **no panel**. Keep the click handler seam clean so Phase 6 can attach the panel to the same `st`-select event.
- **Mobile bottom-sheet, info "sГ¶gulegt, ekki spГЎ" panel, full loading/empty/no-data polish вҖ” Phase 7.** Here the list + legend degrade to a **functional toggle** on narrow screens only; no polished sheet chrome.
- **Reverse "worst weather" ranking (RANK-04), adjustable weight sliders (WGT-01)** вҖ” deferred (v1.x / v2). Do not build.

## Color

**Existing Phase 3/4 color roles reused verbatim.** The ONLY new tokens are the score ramp. Accent red (`--accent`) **remains reserved to the temperature numeral** вҖ” the score ramp is a chromatically distinct blueвҶ’tealвҶ’green family and **must not** reuse or approach the accent red.

### New tokens вҖ” sequential score-color ramp (BuGn вҖ” RECONCILED in 05-02)

> **RECONCILIATION NOTE (05-02, superseding the original slate table):** the authoritative ramp
> shipped in code is the **ColorBrewer BuGn** ramp (cool light blue вҶ’ vivid dark green), pinned by
> `scoreColor()` (`site/src/map/score-color.ts`) + its boundary tests in 05-01. The earlier
> 11-stop **slate вҶ’ yellow-green** table below (`#5B6B7A вҖҰ #84A81F`) is **SUPERSEDED and NOT used** вҖ”
> both the legend swatches AND the per-marker color read from the BuGn `--score-*` tokens in
> `tokens.css`, which are `scoreColor()` sampled at each integer stop. The values below are the
> **shipped BuGn tokens**.

A **colorblind-safe SEQUENTIAL ramp** keyed to the 0вҖ“10 score. Low = pale cool blue; high = vivid dark green ("gott veГ°ur = grГҰnt"). It is **NOT** a redвҖ“green diverging scale (accent red is reserved AND redвҖ“green is colorblind-hostile). The ramp is **monotonic in luminance** (lightвҶ’dark as score rises), so it survives grayscale вҖ” a built-in backup for the color-not-sole-channel rule вҖ” and moves along a blueвҶ’tealвҶ’green hue axis that protanopia/deuteranopia/tritanopia all preserve.

| Token | Score | Hex (shipped, BuGn) | Notes |
|-------|-------|---------------------|-------|
| `--score-0` | 0 | `#edf8fb` | pale blue (low) вҖ” needs the --hairline floor to read (Pitfall 3) |
| `--score-1` | 1 | `#ddf2f1` | |
| `--score-2` | 2 | `#ccece6` | |
| `--score-3` | 3 | `#b3e2d8` | |
| `--score-4` | 4 | `#99d8c9` | teal |
| `--score-5` | 5 | `#80cdb7` | |
| `--score-6` | 6 | `#66c2a4` | |
| `--score-7` | 7 | `#49b282` | |
| `--score-8` | 8 | `#2ca25f` | green |
| `--score-9` | 9 | `#168846` | |
| `--score-10` | 10 | `#006d2c` | vivid dark green (highest вҖ” "grГҰnt") |

- **Colorblind-safety:** BuGn is a published ColorBrewer colorblind-safe sequential scheme; it is monotonic in luminance so under any dichromatic simulation the ordering is still readable as a light/dark gradient, and its blueвҶ’tealвҶ’green hue axis never crosses the redвҖ“green confusion axis. The reserved temperature red `#C0392B` is hue-distinct (warm red vs cool blueвҖ“green) from every ramp stop вҖ” a marker's red temp numeral can never be confused with its score bar.
- **Pale-low-end mitigation (Pitfall 3):** because the BuGn low end is near-white, the marker color is applied as a **4вҖ“6px left color bar** (not a thin 2px ring) over a 1px `--hairline` floor, and the legend ramp bar carries a 1px `--hairline` border вҖ” so `--score-0/1` still read as an intentional colored edge.
- **Score binning:** the marker **interpolates** the continuous 0вҖ“10 score via `scoreColor()`; the legend swatches use the same `scoreColor()`-sampled `--score-*` tokens, so the legend and marker agree exactly at the labeled stops.

### How the ramp applies (marker)

The neutral Phase 3/4 white pill becomes score-colored **on two channels, never overwriting the white body or the red temperature**:

| Channel | Spec (RECONCILED in 05-02) |
|---------|------|
| **Score left-bar** | The scored pill keeps its 1px `--hairline` box border (the floor) and gains a **4вҖ“6px LEFT COLOR BAR in the station's score-ramp color** (`var(--pill-score)` = `scoreColor(datum.score)`), plus an inset 1px `--hairline` outline. *(A thin 2px full ring was rejected: the pale BuGn low end lacks contrast as a hairline ring вҖ” Pitfall 3. A chunky left bar reads against both the white pill and the muted `--dominant` basemap.)* The white body, red temp, ink wind, and shadow are unchanged. Perceptible as an essential boundary because it is paired with the numeric badge (never the sole channel). |
| **Score badge** | A small chip **prepended** to the pill (leading edge, before the temperature). The badge shows the **numeric score** (`7,8`) at Compact-value 13px Semibold, **ink `#1F2933` on a white chip** (13:1 вҖ” legibility never depends on the ramp), with a 2px `var(--pill-score)` ring so it doubles as a color key. **The numeral is the required non-color channel; the ring color is the redundant channel.** |

> **Why the numeral is on white, not on the ramp fill:** a filled-ramp badge would force a per-stop text-color crossover (white text legible only for scores 0вҖ“6, ink text only for 7вҖ“10, with score 6 marginal either way). Keeping the numeral ink-on-white makes score legibility ramp-independent and AA-safe at every stop. The ramp does its job as the **ring** channel on both the pill and the badge.

### `ГіfullnГҰgjandi gГ¶gn` and `ГЎn Гәrkomu` on the ramp

| Case | Datum condition | Marker treatment |
|------|-----------------|------------------|
| **Insufficient (`score:null`, unscorable)** | `sufficient === false` OR `score === null` (domain: `contributing` empty) | **Existing Phase 3 muted pill** (`--marker-empty` tokens) вҖ” **NO score ring, NO score badge, OFF the ramp entirely.** Renders `ГіfullnГҰgjandi gГ¶gn`. **Excluded from the ranked list.** |
| **Scored "ГЎn Гәrkomu"** | `score !== null` AND `missingRain === true` (temp+wind renormalized, rain absent) | **Fully colored + badged + ranked** on the renormalized score. Carries the existing "ГЎn Гәrkomu" signal (precip glyph omitted per Phase 3) AND an **"ГЎn Гәrkomu" badge** in the ranked row. It is NOT muted вҖ” a rain-less station is scored fairly, not penalized. |

> Domain note for the planner: `missingRain === true` does **not** imply `score === null`. A station can be `missingRain: true` **and** scored (ГЎn Гәrkomu) вҖ” colored + ranked. Only an **empty `contributing`** (вҮ’ `score === null`) is the off-scale/excluded case. Extend `MarkerDatum` (currently lacking score fields) to carry `score`, `contributing`/`missingRain` from `combine()`; do NOT recompute the math.

### Existing roles reused

| Role | Value (existing token) | Usage in this phase's panels |
|------|------------------------|------------------------------|
| Dominant (60%) | `--dominant` `#E8EBED` | Ranked-row hover/selected fill; legend track backdrop |
| Secondary (30%) | `--secondary` `#FFFFFF` | Legend + ranked-panel surface at `rgba(255,255,255,0.92)` + `backdrop-filter: blur(8px)` вҖ” **identical treatment to the Phase 3 header + Phase 4 control bar** so all chrome reads as one design language |
| Ink | `--ink` `#1F2933` | Panel titles, station names, score numerals (badge + row), focus outline (`2px solid var(--ink)`, offset 2px) |
| Muted ink | `--muted-ink` `#5B6670` | Rank numerals, "ГЎn Гәrkomu" badge, empty-state line, legend endpoint captions |
| Marker empty | `--marker-empty-*` | Reused unchanged for the `ГіfullnГҰgjandi gГ¶gn` off-scale pill |
| Hairline | `--hairline` `rgba(31,41,51,0.10)` | Panel borders, row separators, legend swatch borders, collapse-toggle border |

**Accent reserved for:** the temperature numeral inside a marker, and nothing else (unchanged). The score ramp is a separate token family and never the accent.

**Destructive:** none вҖ” this phase has no destructive actions. Selecting a station (row click) is non-destructive, instantly reversible, and URL-encoded (`st`). No confirmations.

**Color is never the sole channel:** every score is accompanied by its **numeral** (badge on the pill, value in the row); the ramp is monotonic in luminance (grayscale-legible); `ГЎn Гәrkomu` carries a **text badge** + the omitted precip glyph; `ГіfullnГҰgjandi gГ¶gn` carries **text** + the muted tone + exclusion from the list. No datum or state is conveyed by hue alone.

## Accessibility Notes

- **Color not the sole channel (score):** every score carries a **numeral** (badge + row value) and the ramp is **monotonic in luminance** (grayscale-legible); `betra`/`verra` captions state the direction in words. The ring color is always redundant to the numeral.
- **Ramp contrast:** the score ring is an essential boundary paired with the numeric badge; it renders at вүҘ2.76:1 against the white pill and вүҘ2.31:1 against the `--dominant` basemap at every stop (verified). The **score numeral** is ink `#1F2933` on a white chip (13:1, AAA) вҖ” legibility is ramp-independent. Ranked-row text is ink-on-white (13:1); muted rank/badge text at 11px on white вүҲ 5.9:1 (AA, unchanged Phase 3 value).
- **List semantics:** the ranked list is an `<ol>` (ordered) inside a labelled `<section aria-label="Bestu staГ°ir">`; each row is a native `<button>` (or `role` equivalent) so it is keyboard-focusable and Enter/Space activates the fly-to/select. Visual `1. 2. 3.` numerals are redundant to the `<ol>` order.
- **Legend + explainer keyboard:** the explainer uses native `<details>`/`<summary>` (Enter/Space toggles, `aria-expanded` free); the collapse toggle is a native `<button>` with `aria-expanded` + `aria-controls`. The legend is a labelled region (`aria-label="SkГҪring ГЎ einkunn"`).
- **Focus:** every interactive element (rows, toggles, disclosure) shows the standard **`2px solid var(--ink)`, offset 2px** focus outline (the Phase 3/4 skeleton). Never accent-red.
- **Marker focus (Phase 6 seam active-ish):** the ranked-row click SELECTS a station and highlights its marker вҖ” author the marker highlight so a Phase 6 focus ring can reuse the same "selected" state without restructuring. Do not add the chart panel.
- **Reduced motion:** the `easeTo` fly-to and any marker-highlight transition respect `prefers-reduced-motion` (instant/minimal). Continues the Phase 3/4 rule.
- **Live region (optional):** when a row selection flies-to a station, an `aria-live="polite"` announcement of the selected station name is acceptable (Claude's discretion) so screen-reader users hear the result of the jump.
- **Tab order:** legend (scale вҶ’ explainer) and ranked panel (collapse toggle вҶ’ rows top-to-bottom) follow visual order; the new panels come after the Phase 4 controls in the document/tab flow, not before the map controls.

## Acceptance-Checkable Visual Criteria (Playwright UI review)

Objective assertions the automated E2E can make against the running preview build:

1. **Markers carry a score-derived color/badge:** at least one rendered marker has a score-ramp ring color (a `--score-*` value / non-`--hairline` border) AND a visible numeric score badge matching `/\d,\d/` (Icelandic comma), distinct from the temperature `В°` value.
2. **Score recolors on selection change:** changing the scrubber/width/year updates at least one marker's score badge value (and/or ring color) **with zero network requests fired** during the interaction (`page.on('request')` count === 0).
3. **Legend renders with a color scale:** a legend region (`aria-label="SkГҪring ГЎ einkunn"` or the `Einkunn` title) is present, containing a rendered color-scale element and the endpoint captions `verra` and `betra`.
4. **Explainer expands showing weight text:** clicking/activating `hvernig er einkunnin reiknuГ°?` sets `aria-expanded="true"` and reveals text containing `Гәrkoma 40%`, `vindur 30%`, and `hiti 30%`.
5. **Ranked panel present + titled:** a `Bestu staГ°ir` panel is rendered with an `<ol>` of rows.
6. **Ranked list rows in descending score order:** the rendered rows' numeric scores are monotonically non-increasing top-to-bottom (each row's parsed score вүӨ the previous row's).
7. **Ranked list updates with selection:** after a selection change the ranked list's contents/order change (at least one row's score value differs), with **zero network requests** fired.
8. **`ГіfullnГҰgjandi` station absent from list:** a station that renders the muted `ГіfullnГҰgjandi gГ¶gn` pill for the current selection does NOT appear as a row in the `Bestu staГ°ir` list.
9. **`ГЎn Гәrkomu` station is ranked + badged:** a station scored `missingRain` (ГЎn Гәrkomu) DOES appear in the ranked list, carrying the `ГЎn Гәrkomu` badge, and its marker is colored (has a score ring/badge), not muted.
10. **Clicking a row moves the map / selects the station:** activating a ranked row changes the map view (center/zoom changes via `easeTo`) AND sets the URL `st` param to that station вҖ” with **no chart panel opened** (Phase 6 seam: assert no panel element appears).
11. **No new fetch for coloring/ranking:** none of the above interactions fire a network request (the coloring + ranking are pure client-side reads of the existing store).
12. **Empty state:** for a selection where no station is scorable, the `Bestu staГ°ir` panel shows the `Engin einkunn` message (not an empty list, no throw/white-screen).
13. **Collapse toggle:** the ranked panel's collapse `<button>` toggles `aria-expanded`, hiding/showing the list; collapsed, it leaves the central map band visible.
14. **Score number format:** every rendered score (badge + rows) matches the Icelandic one-decimal comma format `/^\d{1,2},\d$/` (e.g. `7,8`, `10,0`), never a `.`-separated value.
