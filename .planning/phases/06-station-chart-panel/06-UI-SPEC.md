# Phase 6 — UI Design Contract

> Visual and interaction contract for the **Station Chart Panel** phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **This spec EXTENDS the Phase 3/4/5 design system** (`site/src/styles/tokens.css`, `markers.css`, `controls.css`, `score.css`, and `03/04/05-UI-SPEC.md`) — it does NOT introduce a new one. All spacing, type roles, surface treatment, focus rules, and existing color roles below are the Phase 3/4/5 tokens reused verbatim. The ONLY new tokens introduced are the **two distribution-chart series tones** (`--chart-temp`, `--chart-wind`) and the **precipitation-bar tone** (`--chart-precip`), plus the panel chrome that consumes them. Accent red (`--accent` `#C0392B`) stays reserved to the temperature numeral on the marker — it is NOT used in the charts. The score BuGn ramp (`--score-*`) is NOT reused for chart series (a chart tone must not be mistaken for a score color).

**Phase scope (contract is for THIS phase only):**
- **Station detail panel** — opens on station select (marker click OR ranked-row click; both already set `stationId` via the Phase 5 seam). Right-side panel; the ranked **"Bestu staðir"** list YIELDS (hides) while the panel is open and RESTORES on close.
- **Distribution candlesticks** for temperature and wind (box = 10th–90th pct, whiskers = min/max, median line) — **statistical distribution semantics, NOT financial OHLC** (no green/red directional coloring).
- **Precipitation bars** — per-day-of-year typical total across qualifying years; honest missing (explicit gap/label, never zero).
- **Daylight readout** — astronomical (lat/lon + doy), Icelandic-labeled, for the period.
- **Missing-data states** — per-chart `engin gögn fyrir þetta tímabil`; án-úrkomu → temp/wind show, precip shows a no-gauge message; whole-station-no-data → panel-level message.
- **Plain-Icelandic reading key** — one mandatory per-chart sentence so the distribution encoding is unmistakable (FEATURES.md candlestick-comprehension risk).
- **ECharts 6** (à-la-carte, lazy-loaded on first panel open) — with a minimal chunk-loading affordance.
- All chart data is computed **client-side from the EXISTING Phase 2 derived files** — the per-(year, doy) daily columns in `DerivedFile.cols` (`t/tx/tn`, `f/fx/fg`, `dv`, `r`) already carry everything the distributions need. **NO pipeline change, NO new fetch shape.**

**Explicitly out of scope (seams noted, not specified here — do NOT build):**
- **Mobile bottom-sheet polish / full responsive chrome — Phase 7.** This phase makes the narrow-screen panel *functional* (panel over map) and structures the panel so Phase 7 can promote it to a draggable bottom sheet; that promotion is a SEAM, not built here.
- **Info "sögulegt meðaltal, ekki spá" panel, attribution/freshness, polished loading/empty/no-data chrome — Phase 7.** This phase ships only the minimal chart-chunk loading affordance and the phase-required no-data messages; the fuller trust/loading chrome is Phase 7.
- **meðaltal / dreifing chart toggle (TOG-01) — v1.x.** The per-doy distribution helper is authored so it can later power the toggle; the toggle UI is NOT built.
- **Station comparison side-by-side (CMP-01) — v1.x.** The panel is a single-station view.

## Color

**Existing Phase 3/4/5 color roles reused verbatim.** The ONLY new tokens are the three chart-series tones. **Accent red (`--accent`) remains reserved to the marker temperature numeral and is NOT used anywhere in the charts.** The score BuGn ramp (`--score-*`) is **NOT reused as a chart series color** — a chart tone must be visually distinct from a score color so a reader never mistakes a temperature box for a "good-score green."

### New tokens — chart series tones (added to `tokens.css`)

Two candlestick tones + one bar tone, chosen to be (a) distinct from each other, (b) distinct from the BuGn score ramp, (c) distinct from the reserved accent red, and (d) **not directionally coded** (no red/green up/down — this is distribution, not finance):

| Token | Value (hex) | Series | Rationale |
|-------|-------------|--------|-----------|
| `--chart-temp` | `#B26A3D` | Temperature candlestick (box fill + median + whiskers) | A muted **warm terracotta/ochre** — reads as "temperature" without being the reserved pure accent red `#C0392B` (browner, lower-chroma, hue-distinct from the accent so the two never collide). |
| `--chart-wind` | `#3D6E8C` | Wind candlestick (box fill + median + whiskers) | A muted **cool steel blue** — reads as "wind/air", clearly separated from the warm temp tone and outside the BuGn green half of the score ramp. |
| `--chart-precip` | `#4A5A6A` | Precipitation bars | A **desaturated slate** — neutral "rain" tone, distinct from both candlestick tones and from every `--score-*` stop (which trend teal→green). |

- Box fill uses the tone at reduced opacity (~0.28–0.35 alpha) so the **median line** (same tone at full opacity, ~1.5–2px) and the whisker/cap strokes (same tone) read on top — the ENCODING is shape/position (box extent, whisker reach, median line), never hue alone.
- **Colorblind-safety:** each chart's meaning is carried by **geometry** (box = 10th–90th band, whiskers = min/max, median line, bar height), which survives any color-vision deficiency and grayscale. Hue only DISTINGUISHES the three series from each other (temp vs wind vs precip) — and each series also carries its own **titled figure** (`Hiti` / `Vindur` / `Úrkoma`) and axis units, so the series is never identified by color alone. The three tones (warm terracotta / cool steel / neutral slate) remain distinguishable under deuteranopia/protanopia/tritanopia simulation (they differ in luminance and warm/cool axis, not on the red–green confusion axis).

### Existing roles reused

| Role | Value (existing token) | Usage in this phase's panel |
|------|------------------------|------------------------------|
| Dominant (60%) | `--dominant` `#E8EBED` | Chart plot-area gridline tone / subtle chart background band; close-button hover fill |
| Secondary (30%) | `--secondary` `#FFFFFF` | Panel surface at `rgba(255,255,255,0.92)` + `backdrop-filter: blur(8px)` — **identical treatment to the Phase 3 header, Phase 4 control bar, Phase 5 legend/ranked panel** so the panel reads as the same design language |
| Ink | `--ink` `#1F2933` | Station name (panel title), chart titles, primary axis prose, daylight value, close-glyph stroke, focus outline (`2px solid var(--ink)`, offset 2px) |
| Muted ink | `--muted-ink` `#5B6670` | Axis tick labels, daylight caption, reading-key prose secondary emphasis, the `engin gögn` / no-gauge message text |
| Hairline | `--hairline` `rgba(31,41,51,0.10)` | Panel border, header ↔ body separator, per-chart figure separators, chart axis lines / gridlines, close-button border |

**Accent reserved for:** the temperature numeral inside a marker, and nothing else (unchanged). The chart temperature series uses `--chart-temp` (terracotta), **never** the accent red.

**Destructive:** none — this phase has no destructive actions. Closing the panel (`store.set({ stationId: null })`) is non-destructive, instantly reversible, URL-encoded (clears `st`). No confirmations.

**Color is never the sole channel:** every chart's meaning is geometry (box/whisker/median/bar) + a titled figure + axis units + a plain-Icelandic reading key. Missing data is TEXT (`engin gögn fyrir þetta tímabil` / no-gauge message), never a blank axis or a flat zero line. The score ramp is not reused, so no chart element can be misread as a score.

| Aspect | Contract |
|--------|----------|
| Placement | `position: fixed; right: 0; z-index: 10;` — the header/control-bar/legend/ranked tier (above markers z5, canvas z0). Top offset `calc(56px + var(--space-lg))` (below the 56px header, matching the ranked panel). Bottom offset `calc(var(--bar-height, 100px) + var(--space-lg))` (clears the control bar, reusing the `--bar-height` var the control bar writes on mount/resize). Surface = `rgba(255,255,255,0.92)` + `blur(8px)` + 1px `--hairline`, radius `8px 0 0 8px` (round only the left corners — hangs off the right edge, matching the ranked panel). |
| Width | ~320–360px on desktop (Claude's discretion within this band). Slightly wider than the 280px ranked list because charts need horizontal room; still within the Phase 4/5 occlusion budget (must leave the central Iceland station band at zoom 6 visible — the map stays visible to the LEFT). |

## Distribution Candlestick Spec (Temperature & Wind)

**Statistical distribution semantics — NOT financial OHLC.** No green/red directional coloring. Both charts share this encoding; only the series tone, axis unit, and reading key differ.

| Element | Spec |
|---------|------|
| X axis | **Day-of-year** across the selected window (`anchorDoy` ± `widthDays`), left → right chronological. Tick labels: short Icelandic dates (`--muted-ink`, 11px, tabular). At the widest window (30 days) thin the tick labels so they don't collide (Claude's discretion — every Nth day). |
| Y axis | Temp chart: **°C**; wind chart: **m/s**. Tick labels 11px `--muted-ink` tabular with the unit implied by the axis (unit shown once, e.g. as an axis name or first tick). Zero line for wind (never below 0). |
| Box | For each day-of-year, across the **qualifying baseline years** (reuse the Phase 1 `qualifyingYears`/`effectiveN` gate — N≥3 / ≥80%), the box spans the **10th → 90th percentile** of that day's values. Fill = series tone at ~0.28–0.35 alpha. |
| Whiskers | Thin caps at **min** and **max** of that day's values, connected to the box by a stalk. Series tone, ~1px stroke. |
| Median line | A horizontal line at the **median** (50th pct) inside the box, series tone at full opacity, ~1.5–2px — the "typical day" read. |
| Colour | Temp = `--chart-temp` (terracotta); Wind = `--chart-wind` (steel blue). **Single tone per chart** — the box is NOT colored up/down by any comparison (that would be the finance semantics to avoid). |
| Percentile method | Linear interpolation between order statistics (Claude's discretion — pick one method and use it consistently; document it in the helper). Small, testable pure helper (`percentile(values, p)`), likely NEW in `@betravedur/domain` or the site data layer. |
| Data source | Computed **client-side** from the station's existing `DerivedFile` — temp from the `t` column (daily mean; `tx`/`tn` available if a min/max variant is preferred, Claude's discretion), wind from the `f` column (daily mean wind speed). Indexed per `(year, doy)` via the existing decode; grouped by leap-folded doy across qualifying years. **NO new fetch, NO pipeline change.** |
| Animation | **Disabled under `prefers-reduced-motion`** (ECharts `animation: false`). Otherwise a brief, minimal enter animation is acceptable. |
| Tooltip (optional, Claude's discretion) | On hover/focus a tooltip may show that day's median / range in Icelandic-formatted numbers. Not required for acceptance; if present it must be keyboard/screen-reader-independent of the a11y summary below. |

### Reading keys (MANDATORY — exact Icelandic sentences)

One plain-Icelandic sentence beneath EACH chart (FEATURES.md flags candlestick comprehension as the #1 risk — the key is mandatory, not optional). These are the final strings:

| Chart | Reading key (final copy) |
|-------|--------------------------|
| Temperature (`Hiti`) | `Kassinn sýnir hitann sem 8 af hverjum 10 dögum lentu í; línan í miðjunni er dæmigerður dagur og strikin sýna kaldasta og hlýjasta dag.` |
| Wind (`Vindur`) | `Kassinn sýnir vindstyrkinn sem 8 af hverjum 10 dögum lentu í; línan í miðjunni er dæmigerður dagur og strikin sýna hægasta og hvassasta dag.` |
| Precipitation (`Úrkoma`) | `Súlurnar sýna dæmigerða úrkomu hvers dags yfir árin; eyða þýðir að úrkoma var ekki mæld, ekki að það hafi verið þurrt.` |

## Precipitation Bar Spec

Precipitation is **bars, not candlesticks** (honors the user's original "bars for rain").

| Element | Spec |
|---------|------|
| Encoding | One **bar per day-of-year**: the **typical (median or mean — Claude's discretion, be consistent) total precipitation** across the qualifying years for that doy. Bar tone = `--chart-precip` (slate). |
| X axis | Same day-of-year axis as the candlestick charts (aligned so the three charts read as one window). |
| Y axis | **mm**, from 0. |
| Honest missing | If the station has **no precip gauge** (AWS / `hasPrecip === false` / `r` column absent) → the WHOLE precip chart is replaced by the **no-gauge message** (see Missing-Data States), NOT a chart of zeros. If SOME days in the window have no qualifying precip data but others do → those days render as an **explicit gap** (no bar) — a gap means "not measured", **never** a zero-height bar (a zero bar would falsely claim "measured, and it was dry"). The reading key states this explicitly. |
| Data source | The `r` column of the existing `DerivedFile`, per `(year, doy)`, across qualifying years. **NO new fetch, NO pipeline change.** |
| Animation | Disabled under `prefers-reduced-motion`. |

## Daylight Readout

| Element | Spec |
|---------|------|
| Computation | **Astronomical**, from the station `lat`/`lon` + the window's day-of-year — a PURE calculation, NO data dependency (works even when the station has no weather data). Small tested helper (hand-rolled sunrise/sunset or a tiny lib — Claude's discretion; if a lib is added it is a second deliberate runtime dep, prefer hand-roll to keep the dep count minimal). |
| Value shown | Daylight **hours** for the period. Show either the range across the window (`from–to klst.`) OR the midpoint day's hours (Claude's discretion; midpoint is simpler and honest for a short window). Icelandic-formatted number (comma decimal), unit `klst.` |
| Label | `Dagsbirta` (Section-title/Body role, ink), value at Compact-value 13px 600 tabular. |
| Placement | Below the three charts, in the scrollable body. A single readout row (label + value), not a chart. |
| Always available | Because it's pure computation, the daylight readout renders **even when all three charts show no-data** — it's the one panel element a data-less station still gets. |

## Missing-Data States

Reuse the Phase 1 honest-coverage contract (N≥3 / ≥80% qualifying years) and the Phase 3/4/5 honesty vocabulary. Three granularities:

| Case | Condition | Panel treatment |
|------|-----------|-----------------|
| **Per-chart no data** | A given metric has no qualifying data for the selected window/years (e.g. temp column empty, or < 3 qualifying years for that metric in-window) | That chart's canvas is replaced by an in-figure message: **`engin gögn fyrir þetta tímabil`** (`--muted-ink`), keeping the chart title above it. **Never** a blank axis, a flat line, or a zero-height render. |
| **Án úrkomu (no gauge)** | Station measures temp/wind but has no precip gauge (`hasPrecip === false` / `r` absent) | Temp + wind charts render normally; the **precip chart** shows a distinct no-gauge message: **`engin úrkomumæling á þessari stöð`** (`--muted-ink`), with the án-úrkomu framing (rain not measured ≠ it was dry). Reuse the Phase 3/5 `án úrkomu` vocabulary. |
| **Whole-station no data** | The station has no qualifying data for ANY metric in the window (all three would be empty) | A **panel-level** message in the body instead of the three empty figures: heading **`Engin gögn`** + body **`Þessi stöð hefur engin gögn fyrir valið tímabil. Prófaðu annað tímabil eða víðara árabil.`** The panel header (station name + close) and the **daylight readout still render** (daylight is data-independent). |

- Messages are TEXT + `--muted-ink`, consistent with the Phase 5 ranked-list empty state. No blank charts, no throws, no white-screen.

| Aspect | Contract |
|--------|----------|

Tone: plain, honest, Icelandic — continues the Phase 3/4/5 voice. The reading keys are deliberately explicit because candlestick comprehension is the known risk.

## Accessibility Notes

- **Charts are `<canvas>` (pixels, opaque to AT) — provide an accessible summary/table fallback.** Each chart `<figure>` carries a `<figcaption>` (or `aria-label`) with a **text summary of the distribution** in Icelandic (e.g. `Hiti: dæmigert bil {p10}–{p90}°C, dæmigerður dagur {median}°C, frá {min}°C til {max}°C yfir tímabilið.`), AND/OR a visually-hidden `<table>` of per-day median/min/max so screen-reader users get the numbers the canvas draws. The reading-key sentence is real DOM text (not baked into the canvas), so AT reads it. The canvas itself gets `role="img"` + an `aria-label` mirroring the summary.
- **Keyboard close:** the close `<button>` is natively focusable (Enter/Space). **Escape** closes the panel when it (or anything inside it) has focus — a keydown handler on the panel calls `store.set({ stationId: null })`.
- **Focus management on open:** when the panel opens, move focus to the panel (the close button or the panel container with `tabindex="-1"`) so keyboard/SR users land in the new content — but do NOT steal focus disruptively from an in-progress map interaction (Claude's discretion on exact target; prefer the close button). Announce the opened station via `aria-live="polite"` (the station name) is acceptable.
- **Focus restoration on close:** on close, return focus to a sensible prior element — the ranked-list row / marker that opened the panel if reachable, else the panel's launching control. Continues the Phase 5 select-seam intent.
- **Focus outline:** every interactive element (the close button, any narrow-screen toggle) shows the standard **`2px solid var(--ink)`, offset 2px** ring (the Phase 3/4/5 skeleton). Never accent-red.
- **Reduced motion:** ECharts `animation: false` (no chart enter/transition animation) AND the chart-chunk loading affordance is static under `prefers-reduced-motion`. Panel open/close uses instant/minimal transition. Continues the Phase 3/4/5 rule.
- **Colorblind-safe:** chart meaning is geometry (box/whisker/median/bar) + titled figures + axis units + reading keys — never hue alone. The three series tones differ in luminance and warm/cool axis (not the red–green confusion axis), and none reuses the score ramp or the accent red.
- **Not a modal (yet):** on desktop the panel is a non-modal side panel (map stays interactive); the ranked list yields but the map, header, and controls remain reachable. Do not trap focus on desktop. (Phase 7 decides bottom-sheet modality on mobile.)
- **Contrast:** panel text is ink-on-white (`--ink` on `rgba(255,255,255,0.92)`, ≥13:1); axis/muted text is `--muted-ink` (~5.9:1 at 11px on white, AA, the established Phase 3 value); the median line and whisker strokes are the full-opacity series tone (each ≥3:1 against the near-white plot area — verify per tone in the plan).

## Acceptance-Checkable Visual Criteria (Playwright UI review)

Objective assertions the automated E2E can make against the running preview build. (Chart internals are canvas — assert on canvas PRESENCE + counts + the surrounding DOM text/aria, not pixel content.)

1. **Clicking a station opens the panel:** clicking a rendered marker (or activating a ranked-list row) makes a `station-panel` element (`aria-label` = the station name) appear that was not present before.
2. **Panel shows temp + wind charts and precip bars:** the open panel contains a `Hiti` figure with a chart `<canvas>`, a `Vindur` figure with a chart `<canvas>`, and a `Úrkoma` figure with a chart `<canvas>` (three chart canvases, titled) — for a station with data.
3. **Reading keys present:** the panel contains the three exact reading-key sentences as DOM text (temp key contains `8 af hverjum 10`; wind key contains `hægasta og hvassasta`; precip key contains `eyða þýðir að úrkoma var ekki mæld`).
4. **Daylight readout present:** the panel contains a `Dagsbirta` label and a value matching `/\d+,\d+\s*klst\./` (Icelandic comma + `klst.`).
5. **No-data station shows the message, not a blank chart:** for a station with no qualifying data in the current window, a chart slot shows `engin gögn fyrir þetta tímabil` (or the panel-level `Engin gögn`) — asserted as TEXT — and does NOT render a misleading populated canvas for that metric.
6. **Án-úrkomu station:** for a station flagged `hasPrecip === false`, the temp + wind charts render (canvases present) AND the precip figure shows `engin úrkomumæling á þessari stöð` (text) instead of a bar canvas.
7. **Close clears selection and restores the ranked list:** activating the close button (`aria-label="Loka"`) removes the `station-panel`, clears the URL `st` param, and makes the Phase 5 `Bestu staðir` ranked list visible again.
8. **Ranked list hidden while panel open:** while the `station-panel` is open, the `Bestu staðir` ranked list is NOT visible (hidden/yielded) — the two right-docked panels are not both shown at rest.
9. **Escape closes the panel:** with focus in the panel, pressing `Escape` closes it (same effect as the close button: panel gone, `st` cleared, ranked list restored).
10. **No new fetch for the panel data:** opening the panel and rendering the charts fires **zero network requests** for chart DATA (the distributions are computed client-side from already-loaded derived data). *(The ECharts JS chunk itself may load once via dynamic import — that JS-chunk request is allowed and distinct from a data fetch; the assertion targets data requests, e.g. to `data/`.)*
11. **Distribution semantics, not finance:** no chart element uses a green-up / red-down directional pairing; the temperature series does NOT use the reserved accent red `#C0392B` and does NOT use a `--score-*` value (guardable via the token names in CSS / ECharts option — grep-gate `--chart-temp`/`--chart-wind`/`--chart-precip` in use, and FAIL on `--accent` or `--score-` inside the chart option module).
12. **Reduced-motion:** under an emulated `prefers-reduced-motion: reduce`, the ECharts instances are created with `animation: false` (assert via the option, or that no chart animation runs).
13. **Number format:** every Icelandic-formatted number rendered as panel DOM text (daylight; any tooltip/summary values in the DOM) uses a comma decimal, never a `.`-separated value.
14. **Accessible summary present:** each chart `<figure>` exposes a text/`aria-label` distribution summary (or a visually-hidden table) so the canvas is not the sole carrier of the data (assert the `figcaption`/`aria-label` / hidden-table text exists).
