# Phase 7 — UI Design Contract

> Visual and interaction contract for the **Responsive UX & Trust States** phase — the last UI phase before ship.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> **This spec EXTENDS the Phase 3/4/5/6 design system** (`site/src/styles/tokens.css`, `markers.css`, `controls.css`, `score.css`, `panel.css`, and `03/04/05/06-UI-SPEC.md`) — it introduces **NO new design system, NO new tokens, and NO new color/type roles**. Every surface, spacing token, type role, focus rule, and honesty-vocabulary string below is the accumulated Phase 3–6 contract reused verbatim. Phase 7's job is to **HARMONIZE** the layout that has accumulated (header, control bar, legend, ranked list, station panel, attribution) into one coherent responsive whole, add the trust (info) panel and the loading/error/empty states, and **pay down the recurring attribution-legibility debt once**.

**Phase scope (contract is for THIS phase only):**
- **Responsive layout (UX-03)** — a single ~640px breakpoint. Desktop keeps the current right-docked **side panel**; phones get a draggable, non-modal **bottom sheet** for the station panel (map stays visible + pannable above it). The ranked list + legend collapse to **chips** on mobile; the control bar stays reachable with no horizontal overflow. Map usable (pan/zoom) at all sizes.
- **Info / trust panel (UX-04)** — a persistent **"i" button** top-right; a panel framing the data as **"sögulegt meðaltal, ekki spá"** (the #1 misconception), how to read it, the Veðurstofan CC BY 4.0 attribution (reusing the domain `ATTRIBUTION` constant), and a **"uppfært {date}"** freshness line from `manifest.json`. **First-visit auto-open** once (localStorage flag), non-blocking for permalink-restored views.
- **Loading / error / empty states (UX-05, incl. accumulated debt)** — an initial **loading affordance**; a **map-load-error** state ("Ekki tókst að hlaða kortið"); an **empty-stations** state ("Engar veðurstöðvar"); and confirmation that the existing Phase 5/6 no-data states stay consistent + un-regressed.
- **The attribution solution (accumulated debt)** — ONE coherent layout where CC BY 4.0 + OSM + Protomaps + Veðurstofa stays fully legible across control-bar + legend + ranked + station-panel + bottom-sheet, desktop AND mobile. This **supersedes** the incremental margin fixes in `controls.css`.

**Explicitly out of scope (do NOT build):**
- **The nightly pipeline / auto-deploy / full national dataset — Phase 8.** This phase only *reads* a freshness timestamp from the manifest; it does not write or automate it. Keep the manifest field as the contract Phase 8 populates.
- **v1.x/v2 features** — station comparison, meðaltal/dreifing toggle, sunshine, adjustable weights, English UI. Deferred.
- **Basemap bilingual label ("ICELAND"/"Ísland")** — low priority; leave unless a trivial Protomaps flavor fix exists (documented limitation).

The score `--score-*` ramp is NOT reused as chrome. The three `--chart-*` tones are NOT reused outside the chart panel.

| Ink | `--ink` `#1F2933` | Info-panel title + prose + the `ekki spá` lead; info "i" glyph stroke; chip labels; map-error + empty-stations headings; the standard `2px solid var(--ink)` focus ring on every new control |

**Color is never the sole channel:** the info button carries a text `aria-label` (`Um kortið`) + the "i" glyph; the map-error and empty states are **TEXT**, never a blank/colored region; the loading affordance is text (`hleð…`) not just a spinner; mobile chips carry text labels, never icon-only-by-color. No state or affordance is conveyed by hue alone. The reserved accent is untouched, so nothing new can be misread as a temperature numeral.

**Single breakpoint: `640px`.** `≥ 640px` = **desktop** (current side-panel layout, unchanged in spirit). `< 640px` = **mobile** (bottom sheet + chips). This matches the existing `@media (max-width: 640px)` blocks already in `controls.css` and `panel.css` — Phase 7 unifies on that one line so the whole app flips coherently.

Region-by-region contract (desktop vs mobile, and how they coexist when the station panel is open on mobile):

| Region | Desktop (≥640px) | Mobile (<640px) | Coexistence when the station panel/sheet is open (mobile) |
|--------|------------------|------------------|-----------------------------------------------------------|
| **Map** (`#map`, z0) | Full-bleed base layer, pan/zoom. | Full-bleed, pan/zoom. Must stay **pannable behind the bottom sheet** (non-modal). | Map fills the area **above** the sheet's peek/expanded top edge; remains interactive (the sheet does not cover the whole viewport and its backdrop is non-modal — see Bottom-Sheet Anatomy). |
| **Header** (`.app-header`, z10) | 56px bar, wordmark + slogan (slogan wraps under wordmark ≤420px, existing rule). | Same 56px bar. | Unaffected — stays at the top edge; the sheet occupies the bottom. |

| **Control bar** (`.control-bar`, z10) | Bottom-fixed, centered inner (max 960px), scrubber track + width buttons + Frá/Til (existing). | Bottom-fixed; the Phase 4 **stepper** replaces the scrubber track, the inner **wraps** (existing `flex-wrap`), width buttons + year selects reflow to ≥44px targets; **no horizontal overflow**. | When the sheet is open, the sheet sits **above** the control bar. At **peek** height the control bar stays visible/reachable at the very bottom edge. Never a permanently-docked panel that covers the map. | While the station sheet is open, the ranked chip stays **collapsed** (its overlay is dismissed) — only one station-focused surface (the sheet) is expanded at a time, echoing the desktop yield. **Z-order (unified, top to bottom):**
`info panel <dialog>` (native top layer / z40) > `map-load-error + loading + empty overlays` (z30) > `bottom sheet` (z20, above the standing chrome so it can rise over the control bar) > `header / control bar / legend / ranked / station-side-panel / chips` (z10) > `markers` (z5) > `map canvas` (z0). The info panel sits ABOVE the sheet (a user can open "Um kortið" while a station sheet is at peek). The error/loading overlays sit above the sheet but below the info panel.

**Occlusion budget (continues Phase 4/5):** on desktop the right panel (≤360px) + bottom-left legend + control bar must leave the central Iceland station band (zoom 6) visible — unchanged. On mobile the standing chrome collapses to chips precisely so the map is not permanently crowded; the sheet at **peek** leaves the majority of the map visible.

## Bottom-Sheet Anatomy (mobile station panel — UX-03)

Promotes the Phase 6 `.station-panel` (`<header>` + independently-scrolling `.station-panel__body` of chart figures) to a draggable sheet **without restructuring the chart stack** (the Phase 6 seam). Google-Maps-style, **non-modal** so the map stays usable.

| Aspect | Contract |
|--------|----------|
| Trigger | Unchanged: `stationId` transitions non-null (marker tap OR ranked-chip row tap, the existing select seam). The sheet mounts/shows at **peek**; the map eases to the station (existing `easeTo`). |
| Snap points | **Two** — **peek** and **expanded**. Peek height `~120px` (`clamp(96px, 18svh, 140px)`) shows the station name (sheet header) + a hint of the first chart, so the user sees *what* they tapped without losing the map. Expanded height `~80svh` (`clamp(70svh, 80svh, 85svh)`) shows the full scrollable chart stack + daylight. A third "full" snap is NOT required. |
| Drag handle | A centered **grabber** bar (≥44px tall hit area, a short rounded `--dominant` bar over `--hairline`) at the sheet top, inside/adjacent to the existing panel `<header>`. Dragging it (pointer/touch) moves the sheet between snaps; releasing snaps to the nearest. A tap on the handle toggles peek↔expanded (a keyboard/tap fallback for the drag). |
| Backdrop | **Non-modal — NO full-screen modal scrim.** The area above the sheet is the live map (pannable/zoomable). Optionally a very light `--dominant` wash only on the sheet's own edge shadow; never a dimming overlay that blocks map interaction. (Contrast with the info panel, which IS modal.) |
| Map stays pannable | The sheet occupies only the lower portion; `pointer-events` pass through above the sheet to the map. Panning the map does not dismiss the sheet. This is an explicit acceptance criterion (map canvas present + pannable at 390px with the sheet open). |
| Close | Dragging below peek, OR a close affordance in the sheet header (the existing `Loka` button, ≥44px), sets `stationId: null` → sheet dismisses, URL `st` clears (existing subscriber), ranked/legend chips restore. **Escape** also closes (keyboard, continues Phase 6). |
| Body scroll | The `.station-panel__body` keeps `overflow-y: auto; min-height: 0` (Phase 6). At **expanded**, the body scrolls the chart stack; at **peek**, the body is clipped to the peek height. Drag-vs-scroll disambiguation: dragging starts from the handle/header; scrolling happens inside the body (Claude's discretion on the exact gesture guard — a common pattern: only the handle/header initiates a sheet drag; the body owns its own scroll). |
| Transitions | Snap animation is a `transform: translateY` transition, **disabled under `prefers-reduced-motion`** (instant snap). No spring/overshoot needed. |
| Reuse | The Phase 6 chart figures, reading keys, daylight readout, no-data states, and lazy-ECharts loading affordance render **unchanged** inside the sheet — Phase 7 only changes the *container* geometry (side panel ↔ sheet), not the content. |

## Info / Trust Panel (UX-04)

The single most important trust surface: it prevents the #1 misconception (forecast vs history) and carries the licensing credit + freshness.

### Info "i" button

| Aspect | Contract |
|--------|----------|
| Placement | A persistent native `<button class="info-button" aria-label="Um kortið">` fixed **top-right**, in the 56px header band, `z-index: 10`, clear of the wordmark and (mobile) the ranked/legend chips. Echoes gottvedur.is/kort's top-right info affordance. |

| Action | Opens the info panel (`<dialog>.showModal()` or equivalent). Always available (does not disappear after first visit — repeat visitors use it to re-read the trust framing). |

### First-visit auto-open (non-blocking)

| Aspect | Contract |
|--------|----------|
| Behavior | On first load, auto-open the info panel **once**, dismissible. Remembered via a localStorage flag (e.g. `bv.infoSeen = "1"`) — a **dismissed-hint boolean only**, not user data/accounts (acceptable per CONTEXT). Repeat visitors just see the "i" button. |
| Permalink guard | The auto-open **must not block a permalink-restored view**. ### Data freshness ({date} from manifest)

| Aspect | Contract |
|--------|----------|
| Source | A **manifest.json** timestamp — **do NOT hardcode**. ## Loading / Error / Empty States (UX-05, incl. accumulated debt)

All states are **TEXT on the glass surface** (never a blank/broken screen), consistent with the Phase 5/6 honesty vocabulary. Same `rgba(255,255,255,0.92)` + blur treatment; `--muted-ink` prose, `--ink` headings; announced to AT via `aria-live` where they appear/disappear dynamically.

| State | Trigger | Placement | Exact Icelandic copy (final) | Behavior |
|-------|---------|-----------|------------------------------|----------|
| **Initial loading** | Boot: while the map style + derived data load (the current blank-until-ready gap) | A lightweight centered affordance over the map area (z30), or a slim top-edge indicator | `hleð…` (muted small; keep it minimal — no heavy spinner chrome per CONTEXT). A static/`aria-live="polite"` line. | Shows on boot; **removed** once the first `renderForState` paints markers (or once the style `load` fires + data resolves). ## The Attribution Solution (accumulated debt — solve once)

**This supersedes the incremental margin fixes in `controls.css`** (`--bar-height` offset, the `60vw` cap, the `.panel-open { margin-right: 344px }` hack). Those were reactive per-surface patches; Phase 7 replaces them with one coherent, robust layout.

### Principle: a **bottom attribution safe zone** + a **canonical full credit in the info panel**

1. **Canonical full credit lives in the info panel** (always modal, never occludable) — the full `ATTRIBUTION.text_is` + modified-data notice + CC BY 4.0 link + OSM + Protomaps. This is the licensing backstop: the complete credit is *always reachable and always legible* via the "i" button, in every viewport and every state.

2. **A minimal always-visible credit on the map** — the MapLibre `AttributionControl` stays (`compact: true`), showing the short credit (CC BY 4.0 + OSM + Protomaps + Veðurstofa) with its native `ⓘ`-expand. It lives in a **reserved bottom safe zone** that no other surface is allowed to paint over.

### The safe zone (the single coherent rule)

| Aspect | Contract |
|--------|----------|
| Reserved band | Define one CSS custom property `--attrib-safe-bottom` = the current bottom offset the attribution must clear. It is the **max** of: the control-bar height, and (mobile) the bottom-sheet peek top. The MapLibre bottom-right/bottom-left controls get `margin-bottom: calc(var(--attrib-safe-bottom) + var(--space-sm))`. This **replaces** the `--bar-height + --space-lg` rule. |
| Horizontal clearance | The attribution is anchored bottom-**right** and constrained so its (possibly wrapped) text stays clear of the bottom-**left** legend column. Instead of the fragile `60vw` cap + right-align hack, reserve the bottom-right corner as attribution-only and keep the legend/legend-chip bottom-**left** with a guaranteed horizontal gap. On desktop with the right station panel open, the attribution reflows to sit **below** the panel's bottom edge (the panel's bottom offset already clears the control bar), OR the attribution's right anchor shifts left of the panel — pick the one that keeps every credit line legible (Claude's discretion, but it must be a *rule*, not a per-surface class toggle). Drop the `.panel-open` body-class hack. |
| Mobile / sheet | When the bottom sheet is open, `--attrib-safe-bottom` tracks the sheet's current top (peek or expanded). At peek, the credit sits just above the peek edge (as in the mobile diagram). At expanded, if the sheet covers the map, the minimal credit may ride at the sheet's top edge OR the info-panel canonical credit is the guarantee — but the compact credit must never be *silently hidden with no legible alternative* (licensing). Prefer: the compact credit reflows above the sheet peek; when expanded fully covers it, the always-reachable info-panel credit satisfies the license. |
| Legibility guarantee | In EVERY state (control bar only / + legend / + ranked / + station side-panel / + bottom sheet, desktop AND mobile) the CC BY 4.0 + OpenStreetMap credit is either directly legible in the safe zone OR one tap away in the info panel with the full text. This is the licensing acceptance bar. |
| Reduced motion | Any safe-zone reflow is layout-only (no animated attribution movement); nothing to animate. |

> Net effect for the executor: **remove** the three attribution hacks in `controls.css`, introduce `--attrib-safe-bottom` (set by the control bar + sheet on mount/resize/drag), and add the info-panel canonical credit. One rule, all states.

## Accessibility Notes

- **Bottom-sheet focus + keyboard:** the sheet is **non-modal** (do NOT trap focus — the map must stay reachable), but it must be **fully keyboard-operable**: the drag handle is a native `<button aria-label="Stækka/minnka spjald"` (or similar) that toggles peek↔expanded on Enter/Space (the keyboard equivalent of the drag); the close (`Loka`) and all chart-figure controls are tab-reachable; **Escape** closes the sheet (continues Phase 6). On open, move focus to the sheet header/close (Phase 6 behavior). Announce the opened station via `aria-live="polite"` (Phase 6).
- **Drag keyboard-operability:** every drag interaction has a non-pointer equivalent — the handle button toggles snaps, and expanded/peek is reachable without a pointer. No functionality is drag-only.
- **Info panel focus/Escape:** native `<dialog>` provides focus-trap + Escape + backdrop-dismiss. Focus enters the dialog on open and returns to the info button on close. The close button carries `aria-label="Loka"` and the standard ink focus ring. (The info panel IS modal — distinct from the non-modal sheet.)
- **Loading/error announced:** the initial loading line is `aria-live="polite"`; the **map-load-error** is `role="alert"` / `aria-live="assertive"` (a failure the user must notice); the empty-stations state is `aria-live="polite"`. States are real DOM text, never a canvas or a color-only cue.
- **Touch targets ≥44px** at mobile sizes: info button, info-panel close, sheet drag handle, sheet close, ranked chip, legend chip, and every control-bar control (existing 44px rules continue). Verified at 390px.
- **Focus outline:** every new interactive element (info button, info-panel close, drag handle, chips) shows the standard **`2px solid var(--ink)`, offset 2px** ring. Never accent-red.
- **Reduced motion:** the sheet snap transition, the info-panel entrance, any chip expand, and the loading affordance are all **static/instant under `prefers-reduced-motion`** (continues the Phase 3–6 rule). The existing `easeTo` fly-to already respects it.
- **Contrast:** all new text is ink-on-white (`--ink` on `rgba(255,255,255,0.92)`, ≥13:1) or muted small (`--muted-ink` ≈ 5.9:1 at 11px on white, AA — the established Phase 3 value). No new low-contrast pairings.
- **Non-regression:** the existing marker/panel/legend/ranked a11y (aria-labels, ordered list semantics, chart aria-summaries) is preserved — the responsive changes are container-geometry only.

## Acceptance-Checkable Visual Criteria (Playwright UI review — MULTIPLE viewports)

Objective assertions the automated E2E makes against the running preview build, driven at **desktop 1280×800** and **mobile 390×844** (Playwright viewport). Assert on DOM/aria/geometry, not pixels.

**Responsive (UX-03):**
1. **Mobile sheet:** at 390px, selecting a station (marker/ranked-row tap) shows the station panel as a **bottom sheet** — a `.station-panel` element positioned at the bottom (its top edge is below the viewport middle at peek), with a drag handle present. At 1280px the same selection shows the **right-docked side panel** (top-below-header, right-aligned), NOT a bottom sheet.
2. **Map pannable with sheet open (390px):** with the sheet open at 390px, the map `<canvas>` is present and the map remains interactive — a programmatic pan (or `map.panBy`) changes `map.getCenter()` while the sheet stays open (non-modal proof).
3. **No horizontal overflow at 390px:** `document.documentElement.scrollWidth <= window.innerWidth` — the control bar (stepper + width + year selects) and chrome do not overflow horizontally.
4. **Touch targets ≥44px (390px):** the info button, sheet close, sheet drag handle, ranked chip, and legend chip each have a rendered box ≥44px in the pressable dimension.
5. **Chips on mobile:** at 390px the ranked list and legend render as **chips** (compact toggles), not permanently-docked panels; tapping the `Bestu staðir` chip reveals the ranked list, tapping `Einkunn` reveals the legend.

**Info / trust panel (UX-04):**
6. **Info button opens the panel:** clicking the top-right info button (`aria-label="Um kortið"`) opens an info panel containing the text `Þetta er sögulegt meðaltal, ekki spá.`, an attribution string containing `CC BY 4.0` and `OpenStreetMap`, and an `uppfært` freshness line — at both 1280px and 390px.
7. **First-visit auto-open once:** on a fresh context (cleared localStorage, no URL params) the info panel is auto-open on load; after dismissing it and reloading (same context), it does **not** auto-open (the localStorage flag suppresses it) while the info button still opens it on demand.
8. **Permalink does not get blocked:** loading a permalink URL (with selection/`st` params) does **not** auto-open a modal that blocks the restored view — the restored selection is present/interactive on load (the info panel is either not auto-opened or instantly dismissible with the view already restored).
9. **Freshness is not hardcoded:** the `uppfært {date}` value reflects the manifest timestamp (assert the rendered date is a valid Icelandic date derived from `manifest.json`, not a fixed literal; if the manifest field is absent the line is omitted, not shown as `Invalid Date`).

**Attribution (licensing — the recurring finding, solved once):**
10. **Attribution legible with panel/sheet open — desktop:** at 1280px with the station side-panel open, the CC BY 4.0 + OpenStreetMap credit is present and **not occluded** (its bounding box does not intersect the panel/legend surfaces, OR the full credit is reachable in the open info panel).
11. **Attribution legible with sheet open — mobile:** at 390px with the bottom sheet at peek, the CC BY 4.0 + OpenStreetMap credit is present and not hidden behind the sheet (its box sits above the sheet's peek top edge), OR reachable via the info panel; the full credit is always reachable via the info button.
12. **No legacy attribution hacks:** the `.panel-open { margin-right: 344px }` rule and the `60vw` cap are removed (grep-gate `panel-open` and `60vw` absent from `controls.css`); the harmonized `--attrib-safe-bottom` rule is present.

**Loading / error / empty (UX-05):**
13. **Initial loading affordance:** on boot before markers paint, a `hleð…` affordance is present over the map area, and it is removed once markers render.
14. **Map-load error:** when the map style/basemap fails to load (simulate a style/tile failure), the overlay shows `Ekki tókst að hlaða kortið` + `Reyndu að hlaða síðunni aftur.` as TEXT with `role="alert"`, and the header + info button stay up.
15. **Empty stations:** when `stations.json` is empty/404, the overlay shows `Engar veðurstöðvar` as TEXT over a rendered basemap (not a blank white screen).
16. **No-data non-regression:** the existing `ófullnægjandi gögn` muted pill, the ranked `Engin einkunn` empty state, and the panel `engin gögn fyrir þetta tímabil` / `Engin gögn` states still render (unchanged strings) — reachable at both viewports (behind the mobile chips where applicable).

**Accessibility:**
17. **Sheet Escape + keyboard:** at 390px with the sheet open, pressing `Escape` closes it (`stationId` cleared, `st` param cleared); the drag handle is a focusable native control that toggles peek↔expanded via keyboard.
18. **Info panel Escape/focus:** the info panel closes on `Escape`, traps focus while open (modal), and returns focus to the info button on close.
19. **Reduced motion:** under emulated `prefers-reduced-motion: reduce`, the sheet snaps instantly and the info-panel/loading affordances use no animation.
