# Phase 3 — UI Design Contract

> Visual and interaction contract for the Static Site Shell & Interactive Map phase.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Reference look: gottvedur.is/kort (white rounded callouts) — but self-hosted PMTiles + OSM, no keys, no vendor branding.

**Phase scope (contract is for THIS phase only):** Icelandic site shell (header + slogan), interactive MapLibre map of Iceland (self-hosted PMTiles), and NEUTRAL station markers (temperature, wind arrow + speed m/s, precipitation indicator) at zoom-appropriate density, plus the CC BY 4.0 / OSM attribution control.

**Explicitly out of scope (seams noted, not specified here):** period/year selectors (Phase 4), score coloring / legend / ranked list (Phase 5), click-to-open chart panel (Phase 6), mobile bottom-sheet / info panel / full trust states (Phase 7).

## Design System

| Property | Value |
|----------|-------|
| Tool | none (vanilla TypeScript + Vite; MapLibre GL JS, not a React component kit) |
| Preset | not applicable — no shadcn/Radix; plain CSS + CSS custom properties for tokens |
| Component library | none — hand-authored DOM/CSS; MapLibre for map + markers |
| Icon library | Inline SVG (hand-authored, ~2 glyphs this phase: wind arrow, precipitation drop). Do NOT pull a heavy icon font — keep the bundle static and small |
| Font | System UI stack: `-apple-system, "Segoe UI", "Helvetica Neue", "Inter", Arial, sans-serif`. Icelandic glyph coverage (ð, þ, æ, ö, á, í, ú) required — system stack and Inter both cover it. No webfont download this phase |

> shadcn gate: not applicable. This is a vanilla-TS/Vite map app (MapLibre-driven canvas + DOM overlays), not a React/Next component project. Tokens are delivered as CSS custom properties in a single `tokens.css`, consumed by both the shell and marker CSS.

## Layout

Full-bleed map with a thin floating header. No sidebar this phase.

| Region | Spec |
|--------|------|
| Map | Full viewport: `position: fixed; inset: 0;` MapLibre canvas fills 100vw × 100dvh. It is the base layer (z-index 0). |
| Header | Slim bar floating over the map top edge. Height 56px. `position: fixed; top: 0; left: 0; right: 0; z-index: 10;`. Semi-opaque secondary surface with a subtle bottom hairline so map stays visible beneath the edges. Contains: wordmark "Betra Veður" (Display weight) + slogan "Leitin að betra veðri" (Label weight, muted) to its right or beneath on narrow widths. |
| Attribution | MapLibre native `AttributionControl`, bottom-right, collapsed-expandable. z-index above canvas, below any future panel. |
| Marker layer | MapLibre symbol layer (preferred) OR DOM marker overlay — see Marker Rendering. Sits above basemap tiles, below header. |

**Seams for later phases (do not build):** a right-edge column is reserved conceptually for the Phase 5 ranked list; a bottom region is reserved for the Phase 4 period selector; leave the map's usable area unobstructed so those can dock without re-layout.

## Spacing Scale

Declared values (multiples of 4, 8-point-friendly):

| Token | Value | Usage |
|-------|-------|-------|
| `--space-xs` | 4px | Gaps inside a marker callout (temp ↔ wind ↔ precip), icon gaps |
| `--space-sm` | 8px | Marker internal padding, header item gaps |
| `--space-md` | 16px | Header horizontal padding, default element spacing |
| `--space-lg` | 24px | Header ↔ content offset, attribution inset |
| `--space-xl` | 32px | Reserved for later-phase panels |
| `--space-2xl` | 48px | Reserved layout gaps |
| `--space-3xl` | 64px | Reserved page-level spacing |

Exceptions:
- Marker minimum interactive/hit target: 44px in the largest dimension (touch-safe seam for the Phase 6 click). Callout visual box may be smaller than 44px tall; pad the hit area to 44px. This is an intentional exception to the 4/8 grid (accessibility touch-target standard).

## Typography

Three roles this phase (well under the 3–4 size / 2 weight cap). Weights: **Regular 400** and **Semibold 600** only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 20px | 600 | 1.2 | Wordmark "Betra Veður" in header |
| Body/Slogan | 14px | 400 | 1.5 | Slogan "Leitin að betra veðri"; attribution text |
| Marker value | 13px | 600 | 1.15 | Temperature and wind-speed numerals inside markers (compact, tabular) |
| Marker unit/label | 11px | 400 | 1.15 | The "m/s" unit and "breytileg átt" / "án úrkomu" fallback text |

- Numerals in markers use `font-variant-numeric: tabular-nums` so values don't jitter at zoom-density changes.
- Temperature is shown as an integer with the degree sign (e.g. `7°`); wind speed as an integer followed by `m/s` (e.g. `5 m/s`). Icelandic locale: no space before `°`, thin/regular space before `m/s`.

## Color

Restrained, cool, Icelandic palette. Muted basemap so the white callouts read as the figure and the land/sea recede as ground — matching the reference. Not a template blue; a desaturated slate-and-stone base with a single warm red reserved for temperature.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#E8EBED` | Map sea/land base tone the PMTiles style resolves toward (muted, low-chroma); page background behind canvas while tiles load |
| Secondary (30%) | `#FFFFFF` | Marker callout surface (white rounded callouts, per reference); header bar surface at `rgba(255,255,255,0.92)` with backdrop blur |
| Accent (10%) | `#C0392B` | **Temperature numeral only.** The single warm signal on the map, exactly as the reference uses red for temperature |
| Ink | `#1F2933` | Primary text: wordmark, wind speed numeral, attribution |
| Muted ink | `#5B6670` | Slogan, unit labels ("m/s"), "breytileg átt", "án úrkomu", attribution secondary |
| Marker empty/missing | `#9AA5AE` on `#F4F6F7` | Muted marker state for a station with no average for the default period ("ófullnægjandi gögn" honesty) — greyed callout, no red |
| Hairline | `rgba(31,41,51,0.10)` | Header bottom border, marker callout border/shadow edge |

Accent reserved for: **the temperature numeral inside a station marker, and nothing else this phase.** Not links, not the header, not the wind arrow, not focus rings. (Score-based coloring in Phase 5 will introduce a separate scale on the marker surface; temperature-red must remain distinguishable from it — a reason to keep red reserved to the numeral now.)

Destructive: none — this phase has no destructive actions.

**Color is never the sole channel:** temperature always carries the `°` glyph + numeral (not just red); wind direction carries an arrow glyph AND a rotation AND the speed numeral; precipitation is an icon shape, not a color. The empty state is distinguished by muted tone AND the absence of values, and (on hover/focus seam) a text label.

## Map Framing & Basemap

| Property | Value |
|----------|-------|
| Center | lat `65.0`, lon `-19.0` (Iceland) |
| Default zoom | `6` (whole-island framing, matching the reference) |
| minZoom / maxZoom | `4` / `12` (12 caps PMTiles extract size per STACK; below 4 Iceland is a dot) |
| maxBounds | Loose Iceland bbox `[-26, 62.5], [-12, 67.5]` so pan can't wander to empty ocean/other continents |
| Basemap | Self-hosted `.pmtiles` via `pmtiles://` protocol (`maplibregl.addProtocol`). No API key anywhere in the client bundle. Style: `protomaps-themes-base` light variant, then desaturate/mute toward `--dominant` so markers dominate |
| Projection | Default Web Mercator (MapLibre default) — no globe this phase |
| Interaction | Pan + zoom (drag, scroll/pinch, double-click zoom) enabled. Rotation/pitch: disable bearing/pitch drag (`dragRotate: false`, `pitchWithRotate: false`) — a flat north-up map reads clearest for a station map |

## Station Marker Anatomy

A white rounded callout ("pill/card") per station, echoing gottvedur.is/kort. NEUTRAL this phase (no score color).

```
 ┌───────────────────────────┐
 │  7°   ↗ 5 m/s   💧         │   ← single compact row
 └───────────────────────────┘
   temp   wind    precip
   (red)  arrow+  indicator
          speed   (omit if "án úrkomu")
```

| Sub-element | Spec |
|-------------|------|
| Callout box | White (`--secondary`), corner radius 8px, padding `4px 8px`, subtle shadow `0 1px 3px rgba(31,41,51,0.18)` + 1px hairline border. Min hit area 44px (padded, transparent). |
| Temperature | Integer °C in accent red, Semibold 13px, tabular. Format `7°`. Always present when an average exists. |
| Wind arrow | Inline SVG arrow glyph, ink color (NOT red). Rotated to the circular-mean direction (`circularMeanDirection().dirDeg`). Arrow points in the direction the wind blows **toward** — document this convention in code; keep it consistent with any later wind rose. 12px glyph. |
| Wind speed | Integer m/s, ink Semibold 13px, tabular, followed by muted `m/s` unit (11px). Format `5 m/s`. |
| Wind — variable | When `circularMeanDirection` returns null OR `resultantSpeed` is ~0 (the atan2(0,0)/near-cancelling case): render no rotated arrow; instead show the muted label **"breytileg átt"** (11px muted) in place of the arrow, keeping the speed numeral if a scalar mean speed exists. |
| Precipitation indicator | Small inline SVG drop/precip glyph, muted ink. Present only when the station has precipitation in the average. |
| Precipitation — "án úrkomu" | When the station scored without rain (the `missingRain` / "án úrkomu" case): **omit the precip glyph entirely.** Do NOT hide the station and do NOT show a zero. The absence of the drop is the signal; on hover/focus (seam) the label "án úrkomu" may appear. |
| Empty / missing average | A station with no qualifying average for the default period renders a **muted callout** (`--marker-empty` tokens): greyed box, a small "—" or the text "ófullnægjandi gögn" (11px muted), no temperature red, no wind arrow. Never crash, never blank. |

Ordering inside the callout, left→right: temperature, then wind (arrow/label + speed), then precipitation. Gaps `--space-xs` (4px).

## Marker Rendering & Zoom Density

**Preferred approach: MapLibre symbol layer with native collision/declutter** — GPU-driven, matches the vector-tile stack, avoids the mobile DOM-marker perf pitfall (research flag). Markers as symbol/text layers let MapLibre's built-in collision detection hide overlapping labels automatically as the user zooms.

Density contract (acceptance-checkable):
- At default zoom (6, whole island), roughly the **major stations** are visible (target ~15, in line with the reference), NOT every station overlapping.
- As the user zooms in, **more stations reveal** (collision frees space); as they zoom out, fewer show. No two callouts should visibly overlap at rest.
- Prioritize which stations survive collision by a stable priority (e.g. station significance / a `symbol-sort-key`), so the same major stations persist rather than flickering arbitrarily.

If a symbol-layer callout proves too constrained to render the composite (temp + arrow + speed + precip) richly, a hybrid is acceptable: symbol layer drives placement/collision, DOM/canvas draws the composite for only the currently-visible (post-collision) subset — never one DOM node per station regardless of zoom. Whichever path: **do not mount hundreds of DOM markers at once.**

## Accessibility Notes

- **Color is not the sole channel** (see Color section): every marker datum has a glyph/number, not just a hue. Passes even for the temperature red — the `°` and numeral carry it.
- **Contrast:** ink `#1F2933` on white callout ≈ 13:1 (AAA). Muted ink `#5B6670` on white ≈ 5.9:1 (AA for the ≥11px labels — acceptable; do not go lighter). Temperature red `#C0392B` on white ≈ 5.3:1 (AA).
- **Focus states seam (Phase 6):** markers are display-only now, but author them as focusable-ready — if a symbol layer, plan the queryable feature id; if DOM, use a `<button>`/`role` skeleton so a visible focus ring (`2px` outline, ink, offset 2px) can be added in Phase 6 without restructuring. Do NOT add click handlers this phase.
- **Reduced motion:** any zoom/pan easing must respect `prefers-reduced-motion` (shorten/disable fly-to animations).
- **Header contrast:** the semi-opaque header must keep the wordmark/slogan at AA over the blurred map beneath (the 0.92 white + blur ensures this; verify over dark map regions).

## Acceptance-Checkable Visual Criteria (Playwright UI review)

These are objective assertions the automated UI review can make against the running site:

1. **Slogan present:** the header contains the exact text `Leitin að betra veðri`.
2. **Wordmark present:** the header contains `Betra Veður`.
3. **Map canvas present:** a MapLibre `canvas.maplibregl-canvas` (or `.maplibre-canvas`) element is rendered and has non-zero width/height.
4. **No API key in bundle:** built JS/HTML contains no `maptiler`/`api_key`/`access_token`/key-like query params (self-hosted PMTiles only).
5. **At least one marker at default zoom:** ≥1 station callout is rendered on initial load at zoom 6.
6. **Marker shows a temperature value:** a rendered marker contains a numeric temperature with the `°` glyph (e.g. matches `/-?\d+°/`).
7. **Marker shows a wind speed:** a rendered marker contains a wind speed with the `m/s` unit (e.g. matches `/\d+\s?m\/s/`) OR the `breytileg átt` label.
8. **Attribution present:** the map contains attribution text naming `Veðurstofa Íslands` and `OpenStreetMap`, and `CC BY 4.0`.
9. **Interactivity:** the map responds to zoom (a zoom-in increases visible marker count OR the map's zoom level changes) — pan/zoom is enabled.
10. **Density sanity:** at default zoom the visible callouts do not exceed a readable count (no wall of overlapping markers — e.g. ≤ ~25 visible callouts) and no two rendered callout bounding boxes fully overlap.
11. **Graceful empty/missing:** a station in the sample with no average for the default period renders a muted callout (or `ófullnægjandi gögn` text), and the page does not throw / white-screen.
